    Spring Boot Configuration Interview Questions

    1. What is the purpose of the @Configuration annotation in Spring Boot?

    Example Answer: The @Configuration annotation in Spring Boot is used to mark a class as a source of bean definitions. It indicates that the class can be used by the Spring IoC container as a source of bean definitions and configuration. This annotation is typically used to define configuration beans that are needed to set up the Spring context.

    Example:

    @Configuration public class AppConfig { @Bean public DataSource dataSource() { return new DataSource(); } }

    2. How does Spring Boot handle property values from application.properties or application.yml files?

    Example Answer: Spring Boot uses the @Value annotation and @ConfigurationProperties to read values from application.properties or application.yml files.

    • @Value Annotation: Allows you to inject a specific property value directly into a field or method.
    @Value("${server.port}") private int serverPort;
    • @ConfigurationProperties Annotation: Maps entire properties or configuration sections to a Java bean. It is useful for grouping related properties together.
    @ConfigurationProperties(prefix = "server") public class ServerProperties { private int port; // getters and setters }

    3. How do you use the @PropertySource annotation in Spring Boot?

    Example Answer: The @PropertySource annotation is used to specify the location of additional property files to be loaded into the Spring environment. It is used in conjunction with @Configuration to include external property files.

    Example:

    @Configuration @PropertySource("classpath:custom.properties") public class AppConfig { @Value("${custom.property}") private String customProperty; }

    4. What is the difference between @Value and @ConfigurationProperties?

    Example Answer:

    • @Value: Injects individual property values into fields or methods. It is useful for simple, single-value injections.
    @Value("${app.name}") private String appName;
    • @ConfigurationProperties: Binds a group of properties to a configuration bean. It is used for more complex configurations where properties are grouped together.
    @ConfigurationProperties(prefix = "app") public class AppProperties { private String name; private String description; // getters and setters }

    5. How can you load different profiles with Spring Boot?

    Example Answer: Spring Boot uses profiles to load different configurations for various environments. Profiles are specified in the application.properties or application.yml files and are activated using the spring.profiles.active property.

    Example:

    • In application.properties:
    spring.profiles.active=dev
    • In application-dev.properties:
    server.port=8081
    • In application-prod.properties:
    server.port=80

    6. How can you access environment variables in Spring Boot?

    Example Answer: Environment variables can be accessed using the @Value annotation or the Environment object.

    • Using @Value:
    @Value("${MY_ENV_VARIABLE}") private String myEnvVar;
    • Using Environment:
    @Autowired private Environment env; public void someMethod() { String myEnvVar = env.getProperty("MY_ENV_VARIABLE"); }

    7. How do you override default properties in Spring Boot?

    Example Answer: You can override default properties in Spring Boot by specifying new values in application.properties, application.yml, or through command-line arguments.

    Example:

    • In application.properties:
    server.port=9090
    • Using Command-Line Arguments:
    java -jar myapp.jar --server.port=9090

    8. What are @Conditional annotations and how are they used?

    Example Answer: @Conditional annotations are used to conditionally include or exclude beans based on certain conditions. They provide a way to create beans only if specific conditions are met.

    Example:

    • @ConditionalOnClass Example:
    @Configuration @ConditionalOnClass(DataSource.class) public class DataSourceConfig { @Bean public DataSource dataSource() { return new DataSource(); } }
    • @ConditionalOnMissingBean Example:
    @Configuration public class DefaultConfig { @Bean @ConditionalOnMissingBean public MyBean defaultBean() { return new MyBean(); } }

    9. How can you use custom property sources in Spring Boot?

    Example Answer: Custom property sources can be used by creating a custom PropertySource or using the @PropertySource annotation.

    Example:

    • Creating a Custom PropertySource:
    public class CustomPropertySource extends PropertySource<String> { public CustomPropertySource(String name) { super(name); } @Override public Object getProperty(String name) { // Custom property retrieval logic return null; } }
    • Using @PropertySource:
    @Configuration @PropertySource("classpath:custom-config.properties") public class CustomConfig { }

    10. What is the role of Environment abstraction in Spring Boot?

    Example Answer: The Environment abstraction in Spring Boot provides access to application properties and environment variables. It allows you to retrieve properties using key-value pairs and supports different property sources.

    Example:

    @Autowired private Environment environment; public void printProperties() { String dbUrl = environment.getProperty("spring.datasource.url"); System.out.println("Database URL: " + dbUrl); }

    11. How do you use @ConfigurationProperties with nested properties?

    Example Answer: @ConfigurationProperties supports nested properties by defining nested classes within your configuration class.

    Example:

    @ConfigurationProperties(prefix = "app") public class AppProperties { private String name; private DatabaseProperties database = new DatabaseProperties(); // getters and setters public static class DatabaseProperties { private String url; private String username; private String password; // getters and setters } }

    12. How do you validate configuration properties using @ConfigurationProperties?

    Example Answer: You can validate configuration properties by using JSR-303/JSR-380 annotations (e.g., @NotNull, @Min, @Max) on fields within your configuration properties class. Additionally, you must add @Validated to your configuration class.

    Example:

    @ConfigurationProperties(prefix = "app") @Validated public class AppProperties { @NotNull private String name; @Min(1) private int maxConnections; // getters and setters } Configuration: @Configuration @Validated public class AppConfig { @Bean @ConfigurationProperties(prefix = "app") public AppProperties appProperties() { return new AppProperties(); } }

    13. What is the ConfigurableEnvironment interface and how is it used?

    Example Answer: ConfigurableEnvironment is an interface that extends Environment and allows modification of the environment properties at runtime. It provides methods to add or modify property sources.

    Example:

    @Autowired private ConfigurableEnvironment environment; public void addCustomPropertySource() { environment.getPropertySources().addLast(new PropertiesPropertySource("myProps", myProperties)); }

    14. How do you use Spring Boot's configuration property file placeholders?

    Example Answer: Spring Boot allows you to use placeholders in property files, which are replaced with actual values during runtime.

    Example:

    server.port=${SERVER_PORT:8080}

    In this example, SERVER_PORT is used if defined, otherwise, the default value 8080 is used.

    15. How can you externalize configuration in Spring Boot?

    Example Answer: You can externalize configuration by using environment variables, command-line arguments, or external property files. Spring Boot will override properties defined in application.properties with values from these sources.

    Example:

    • Environment Variables:
    export SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/mydb
    • Command-Line Arguments:
    java -jar myapp.jar --spring.datasource.url=jdbc:mysql://localhost:3306/mydb
    • External Property File:
    spring.config.location=classpath:/default.properties,classpath:/override.properties

    16. How can you programmatically access and manipulate configuration properties in Spring Boot?

    Example Answer: You can programmatically access and manipulate configuration properties using the Environment interface or by autowiring configuration beans.

    Example:

    • Using Environment:
    @Autowired private Environment environment; public void printProperties() { String dbUrl = environment.getProperty("spring.datasource.url"); System.out.println("Database URL: " + dbUrl); }
    • Using @ConfigurationProperties Bean:
    @Component @ConfigurationProperties(prefix = "app") public class AppProperties { private String name; private int maxConnections; // getters and setters } @Autowired private AppProperties appProperties; public void displayProperties() { System.out.println("App Name: " + appProperties.getName()); System.out.println("Max Connections: " + appProperties.getMaxConnections()); }

    17. What is the @Profile annotation and how is it used?

    Example Answer: The @Profile annotation is used to conditionally create beans based on the active profiles. It allows you to specify which beans should be instantiated based on the profile that is active.

    Example:

    @Configuration @Profile("development") public class DevConfig { @Bean public DataSource devDataSource() { return new HikariDataSource(); // Development datasource } } @Configuration @Profile("production") public class ProdConfig { @Bean public DataSource prodDataSource() { return new HikariDataSource(); // Production datasource } }

    Activating Profiles:

    • In application.properties:
    spring.profiles.active=development
    • Or as a command-line argument:
    java -jar myapp.jar --spring.profiles.active=production

    18. How can you manage multiple configuration files in Spring Boot?

    Example Answer: You can manage multiple configuration files by specifying their locations and profiles in application.properties or application.yml. Spring Boot supports hierarchical configurations.

    Example:

    • Using application.properties:
    spring.config.name=myapp spring.config.location=classpath:/config/,classpath:/moreconfig/
    • Using Profiles:
    # application.properties spring.profiles.active=dev # application-dev.properties server.port=8081 # application-prod.properties server.port=80

    19. How can you use @PropertySource to load properties from a custom file?

    Example Answer: You can use @PropertySource to specify a custom property file location.

    Example:

    @Configuration @PropertySource("classpath:custom-config.properties") public class CustomConfig { @Value("${custom.property}") private String customProperty; @Bean public String customProperty() { return customProperty; } }

    20. How can you use environment-specific property files in a Spring Boot application?

    Example Answer: Spring Boot allows you to use environment-specific property files by naming convention and profile configuration.

    Example:

    • application-dev.properties:
    server.port=8081
    • application-prod.properties:
    server.port=80

    Activating Profiles:

    • In application.properties:
    spring.profiles.active=dev
    • As a command-line argument:
    java -jar myapp.jar --spring.profiles.active=prod

    21. How do you use @PropertySources to include multiple property files?

    Example Answer: You can use the @PropertySources annotation to include multiple property files.

    Example:

    @Configuration @PropertySources({ @PropertySource("classpath:application.properties"), @PropertySource("classpath:extra.properties") }) public class AppConfig { @Value("${some.property}") private String someProperty; @Bean public String someProperty() { return someProperty; } }

    22. How can you manage configuration for different deployment environments (e.g., development, staging, production)?

    Example Answer: You can manage configurations for different environments by using profiles and environment-specific property files.

    Example:

    • application-dev.properties for development.
    • application-prod.properties for production.

    Activating Profiles:

    • In application.properties:
    spring.profiles.active=dev
    • Command-Line Argument:
    java -jar myapp.jar --spring.profiles.active=prod
    • Using Profiles in Beans:
    @Configuration @Profile("dev") public class DevConfig { // Bean definitions specific to development }

    23. How do you reload configuration properties at runtime in a Spring Boot application?

    Example Answer: To reload configuration properties at runtime, you can use Spring Cloud’s @RefreshScope annotation if you’re using Spring Cloud Config.

    Example:

    @RefreshScope @Component public class RefreshableBean { @Value("${some.property}") private String someProperty; // Methods using someProperty }

    Triggering Refresh: Send a POST request to /actuator/refresh to reload the configuration.

    24. Using @PropertySource and @PropertySources

    You can conditionally load properties from custom property files using the @PropertySource annotation.

    Example:

    @Configuration @PropertySource("classpath:custom.properties") public class CustomConfig { // Beans and configuration }

    Example with Multiple Property Sources:

    @Configuration @PropertySources({ @PropertySource("classpath:application.properties"), @PropertySource("classpath:extra.properties") }) public class AppConfig { // Beans and configuration }

    25. Disabling Auto-Configuration

    Spring Boot allows you to disable specific auto-configuration classes.

    Example:

    • In application.properties:
    spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
    • As a Command-Line Argument:
    java -jar myapp.jar --spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration

    Disabling Specific Auto-Configuration Classes Programmatically:

    @SpringBootApplication(exclude = {DataSourceAutoConfiguration.class}) public class MyApplication { public static void main(String[] args) { SpringApplication.run(MyApplication.class, args); } }

    26. Using Environment to Programmatically Enable or Disable Beans

    You can use the Environment object to access property values and conditionally enable or disable beans based on these values.

    Example:

    @Configuration public class AppConfig { @Autowired private Environment environment; @Bean public SomeService someService() { String featureEnabled = environment.getProperty("app.feature.enabled", "false"); if ("true".equals(featureEnabled)) { return new SomeService(); } else { return null; // or return a default implementation } } }

    27. How can you use the @Value annotation to set default values for properties?

    Example Answer: The @Value annotation allows you to specify default values directly in the annotation. This is useful for injecting single property values with fallback options.

    Example:

    @Component public class MyComponent { @Value(${app.name:DefaultAppName}) private String appName; @Value(${app.timeout:30}) private int timeout; // getters and methods }

    In this example, if app.name or app.timeout is not defined in the configuration, "DefaultAppName" and 30 will be used as defaults.

    28. How do you set default values for properties using YAML configuration?

    Example Answer: In YAML configuration files, default values are set similarly to properties files. You define the default values directly in the YAML configuration.

    Example:

    app: name: DefaultAppName timeout: 30

    When using this YAML configuration, if these properties are not overridden by other configurations or profiles, the default values DefaultAppName and 30 will be applied.

    29. How can you set default values for properties in a Spring Boot @Bean configuration method?

    Example Answer: You can set default values directly within a @Bean method if certain properties are not defined in the configuration.

    Example:

    @Configuration public class AppConfig { @Bean public MyService myService(@Value(${app.name:DefaultAppName}) String appName, @Value(${app.timeout:30}) int timeout) { return new MyService(appName, timeout); } }

    In this example, if app.name or app.timeout is not defined, the default values "DefaultAppName" and 30 will be used when creating the MyService bean.
