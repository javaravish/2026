    Spring REST FAQ

    1. What is the purpose of @RestController in Spring?

    Answer: @RestController is a specialized version of @Controller in Spring that combines @Controller and @ResponseBody. It is used to create RESTful web services. Methods in a @RestController return data directly, rather than views, and the data is automatically serialized into JSON or XML.

    Example:

    @RestController
    @RequestMapping("/api")
    public class MyRestController {

      @GetMapping("/greeting")
      public Map<String, String> getGreeting() {
        Map<String, String> response = new HashMap<>();
        response.put("message", "Hello, World!");
        return response;
      }
    }

    In this example, the getGreeting method returns a JSON response with a greeting message.

    2. How do you handle HTTP methods in Spring REST?

    Answer: Spring REST uses method-specific annotations to handle different HTTP methods. These annotations map HTTP requests to specific controller methods.

    • @GetMapping: Handles HTTP GET requests.
    • @PostMapping: Handles HTTP POST requests.
    • @PutMapping: Handles HTTP PUT requests.
    • @DeleteMapping: Handles HTTP DELETE requests.
    • @PatchMapping: Handles HTTP PATCH requests.

    Example:

    @RestController
    @RequestMapping("/api/users")
    public class UserController {

      @GetMapping("/{id}")
      public User getUser(@PathVariable("id") Long id) {
        // Retrieve user by ID
        return userService.findById(id);
      }

      @PostMapping
      public User createUser(@RequestBody User user) {
        // Create a new user
        return userService.save(user);
      }

      @PutMapping("/{id}")
      public User updateUser(@PathVariable("id") Long id, @RequestBody User user) {
        // Update user with ID
        return userService.update(id, user);
      }

      @DeleteMapping("/{id}")
      public void deleteUser(@PathVariable("id") Long id) {
        // Delete user by ID
        userService.delete(id);
      }
    }

    3. What is @RequestBody used for in Spring REST?

    Answer: @RequestBody is used to bind the HTTP request body to a method parameter in a controller. It is typically used to handle the data sent in the body of POST, PUT, or PATCH requests.

    Example:

    @PostMapping("/users")
    public ResponseEntity<User> createUser(@RequestBody User user) {
      User createdUser = userService.save(user);
      return new ResponseEntity<>(createdUser, HttpStatus.CREATED);
    }

    Here, @RequestBody binds the JSON data in the request body to the User object.

    4. What is @ResponseStatus and how is it used?

    Answer: @ResponseStatus is used to specify the HTTP status code that should be returned for a particular response. It is often used in custom exceptions to return specific HTTP status codes.

    Example:

    @ResponseStatus(HttpStatus.NOT_FOUND)
    public class UserNotFoundException extends RuntimeException {
      public UserNotFoundException(String message) {
        super(message);
      }
    }

    In this example, when UserNotFoundException is thrown, a 404 Not Found status code is returned.

    5. How do you handle exceptions globally in Spring REST?

    Answer: You can handle exceptions globally in Spring REST using @ControllerAdvice and @ExceptionHandler. @ControllerAdvice is used to define global exception handling logic, while @ExceptionHandler methods handle specific exceptions.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(UserNotFoundException.class)
      public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex) {
        ErrorResponse errorResponse = new ErrorResponse("User not found", ex.getMessage());
        return ResponseEntity.ok(errorResponse);
      }

      @ExceptionHandler(Exception.class)
      public ResponseEntity<ErrorResponse> handleGlobalException(Exception ex) {
        ErrorResponse errorResponse = new ErrorResponse("Internal server error", ex.getMessage());
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(errorResponse);
      }
    }

    ErrorResponse:

    public class ErrorResponse {
      private String error;
      private String message;

      // Constructors, getters, and setters
    }

    6. What is @CrossOrigin and how is it used?

    Answer: @CrossOrigin is used to handle Cross-Origin Resource Sharing (CORS) in Spring REST applications. It allows you to specify which origins are permitted to access resources from your server.

    Example:

    @RestController
    @CrossOrigin("http://example.com")
    public class MyRestController {

      @GetMapping("/data")
      public Map<String, String> getData() {
        Map<String, String> response = new HashMap<>();
        response.put("key", "value");
        return response;
      }
    }

    In this example, requests from http://example.com are allowed to access the /data endpoint.

    7. How do you enable HATEOAS (Hypermedia As The Engine Of Application State) in Spring REST?

    Answer: HATEOAS is supported in Spring REST using the spring-hateoas library. You can use RepresentationModel and EntityModel to include links in your responses.

    Example:

    @RestController
    @RequestMapping("/api/users")
    public class UserController {

      @GetMapping("/{id}")
      public EntityModel<User> getUser(@PathVariable("id") Long id) {
        User user = userService.findById(id);
        EntityModel<User> resource = EntityModel.of(user);
        WebMvcLinkBuilder linkToUsers = linkTo(methodOn(UserController.class).getAllUsers());
        resource.add(linkToUsers.withRel("all-users"));
        return resource;
      }

      @GetMapping
      public CollectionModel<EntityModel<User>> getAllUsers() {
        List<User> users = userService.findAll();
        List<EntityModel<User>> userResources = users.stream()
          .map(user -> EntityModel.of(user,
            linkTo(methodOn(UserController.class).getUser(user.getId())).withSelfRel())
        .collect(Collectors.toList());
        return CollectionModel.of(userResources,
          linkTo(methodOn(UserController.class).getAllUsers()).withSelfRel());
      }
    }

    In this example, EntityModel and CollectionModel are used to include HATEOAS links in the response.

    8. What is the @RequestPart annotation and how is it used?

    Answer: @RequestPart is used to handle multipart file uploads where files and other parts of the request can be processed separately.

    Example:

    @PostMapping("/upload")
    public ResponseEntity<String> handleFileUpload(@RequestPart("file") MultipartFile file,
        @RequestPart("metadata") String metadata) {
        // Process file and metadata
        return ResponseEntity.ok("File and metadata received");
    }

    Here, @RequestPart allows you to handle both file uploads and additional request parts.

    10. How do you implement pagination and sorting in Spring REST?

    Answer: Pagination and sorting can be implemented using Pageable and Sort from Spring Data.

    Example:

    @GetMapping("/users")
    public Page<User> getUsers(@RequestParam int page, @RequestParam int size, @RequestParam String sort) {
      Pageable pageable = PageRequest.of(page, size, Sort.by(sort));
      Page<User> usersPage = userRepository.findAll(pageable);
      return usersPage;
    }

    Here, Pageable and Sort are used to handle pagination and sorting, and the method returns a Page of users.

    11. What is @RequestParam and how is it used in Spring REST?

    Answer: @RequestParam is used to extract query parameters from the URL and bind them to method parameters in a controller. It is commonly used in GET requests to handle parameters sent as part of the query string.

    Example:

    @RestController
    @RequestMapping("/api")
    public class UserController {

      @GetMapping("/search")
      public List<User> searchUsers(@RequestParam(name = "name", required = false) String name,
        @RequestParam(name = "age", required = false) Integer age) {
          return userService.search(name, age);
      }

    }

    In this example, @RequestParam is used to bind the name and age query parameters to the method parameters.

    12. How do you handle versioning in Spring REST APIs?

    Answer: Versioning can be handled in several ways in Spring REST APIs:

    • URI Versioning: Include the version in the URL path.
    @RestController
    @RequestMapping("/api/v1/users")
    public class UserControllerV1 {
      // Methods for version 1
    }

    @RestController
    @RequestMapping("/api/v2/users")
    public class UserControllerV2 {
      // Methods for version 2
    }
    • Request Parameter Versioning: Include the version as a query parameter.
    @RestController
    @RequestMapping("/api/users")
    public class UserController {

      @GetMapping(params = "version=1")
      public String getUsersV1() {
        return "Version 1";
      }

      @GetMapping(params = "version=2")
      public String getUsersV2() {
        return "Version 2";
      }

    }
    • Header Versioning: Include the version in the request headers.
    @RestController
    @RequestMapping("/api/users")
    public class UserController {

      @GetMapping
      public String getUsers(@RequestHeader("API-Version") String apiVersion) {
        if ("1".equals(apiVersion)) {
          return "Version 1";
        } else if ("2".equals(apiVersion)) {
          return "Version 2";
        }
        return "Unknown version";
      }

    }

    @Configuration

    14. What is the role of @ResponseBody in a Spring REST controller?

    Answer: @ResponseBody is used to indicate that the return value of a method should be written directly to the HTTP response body, rather than being interpreted as a view name. It is typically used in REST controllers to return data (e.g., JSON or XML) instead of a view.

    Example:

    @RestController
    public class MyRestController {

      @GetMapping("/data")
      @ResponseBody
      public Map<String, String> getData() {
        Map<String, String> response = new HashMap<>();
        response.put("key", "value");
        return response;
      }

    }

    In this example, @ResponseBody ensures that the getData method returns the data as JSON.

    15. How can you use Spring REST to consume a third-party REST API?

    Answer: You can use RestTemplate or the WebClient (from Spring WebFlux) to consume third-party REST APIs.

    Using RestTemplate:

    @Service
    public class ApiService {

      private final RestTemplate restTemplate;

      @Autowired
      public ApiService(RestTemplate restTemplate) {
        this.restTemplate = restTemplate;
      }

      public String getExternalData() {
        String url = "https://api.example.com/data";
        return restTemplate.getForObject(url, String.class);
      }

    }

    Using WebClient:

    @Service
    public class ApiService {

      private final WebClient webClient;

      @Autowired
      public ApiService(WebClient.Builder webClientBuilder) {
        this.webClient = webClientBuilder.baseUrl("https://api.example.com").build();
      }

      public Mono<String> getExternalData() {
        return webClient.get()
          .uri("/data")
          .retrieve()
          .bodyToMono(String.class);
      }

    }

    16. How can you test Spring REST controllers?

    Answer: You can test Spring REST controllers using MockMvc for integration tests or @WebMvcTest for focused tests.

    Integration Test Example with MockMvc:

    @SpringBootTest
    @AutoConfigureMockMvc
    public class UserControllerTests {

      @Autowired
      private MockMvc mockMvc;

      @Test
      public void testGetUser() throws Exception {
        mockMvc.perform(get("/api/users/1"))
          .andExpect(status().isOk())
          .andExpect(content().json("{\"id\":1,\"name\":\"John Doe\"}"));
      }

    }

    Focused Test Example with @WebMvcTest:

    @WebMvcTest(UserController.class)
    public class UserControllerTests {

      @Autowired
      private MockMvc mockMvc;

      @MockBean
      private UserService userService;

      @Test
      public void testGetUser() throws Exception {
        User user = new User();
        user.setId(1L);
        user.setName("John Doe");
        Mockito.when(userService.findById(1L)).thenReturn(user);

        mockMvc.perform(get("/api/users/1"))
          .andExpect(status().isOk())
          .andExpect(content().json("{\"id\":1,\"name\":\"John Doe\"}"));
      }

    }

    17. How do you secure REST APIs in Spring?

    Answer: You can secure REST APIs in Spring using Spring Security. Common practices include:

    • Basic Authentication: Use HTTP Basic Authentication for simple scenarios.
    • OAuth2 / JWT: Implement OAuth2 or JSON Web Tokens (JWT) for more secure, token-based authentication.

    Basic Authentication Example:

    @Configuration
    @EnableWebSecurity
    public class SecurityConfig extends WebSecurityConfigurerAdapter {

      @Override
      protected void configure(HttpSecurity http) throws Exception {
        http
          .authorizeRequests()
            .anyRequest().authenticated()
          .and()
          .httpBasic();
      }

    }

    JWT Example:

    @Configuration
    @EnableWebSecurity
    public class SecurityConfig extends WebSecurityConfigurerAdapter {

      @Override
      protected void configure(HttpSecurity http) throws Exception {
        http
          .authorizeRequests()
            .anyRequest().authenticated()
          .and()
          .csrf().disable()
          .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
      }

      @Bean
      public JwtFilter jwtFilter() {
        return new JwtFilter();
      }

    }

    18. What is the purpose of @RequestParam in Spring MVC?

    19. What is the difference between @PathVariable and @RequestParam?

    Answer: @PathVariable and @RequestParam are used to extract data from the URL, but they are used in different contexts:

    • @PathVariable is used to extract values from URI template variables (i.e., path segments). It's useful for RESTful URLs where parameters are part of the path.
    • @RequestParam is used to extract query parameters from the URL. It's useful for parameters passed in the query string.

    Example with @PathVariable:

    @RestController
    @RequestMapping("/api")
    public class MyController {

      @GetMapping("/greet/{name}")
      public String greetUser(@PathVariable("name") String name) {
        return "Hello, " + name;
      }

    }

    Example with @RequestParam:

    @RestController
    @RequestMapping("/api")
    public class MyController {

      @GetMapping("/greet")
      public String greetUser(@RequestParam("name") String name) {
        return "Hello, " + name;
      }

    }

    20. How does Spring handle exception handling in REST APIs?

    Answer: Spring provides various mechanisms for exception handling in REST APIs, including:

    • @ExceptionHandler: Handle exceptions within a specific controller.
    • @ControllerAdvice: Define global exception handling for all controllers.
    • ResponseEntityExceptionHandler: Extend this class for custom exception handling across the application.

    Example with @ExceptionHandler:

    @RestController
    public class MyController {

      @GetMapping("/data")
      public String getData() {
        throw new RuntimeException("Error occurred");
      }

      @ExceptionHandler(RuntimeException.class)
      public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("An error occurred: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
      }

    }

    Example with @ControllerAdvice:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(RuntimeException.class)
      public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("An error occurred: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
      }

    }
