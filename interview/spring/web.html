    Spring MVC Interview Questions and Answers

    1. What is Spring MVC?

    Answer: Spring MVC is a framework that helps in building web applications. It follows the Model-View-Controller (MVC) architectural pattern to separate the application’s concerns:

    • Model: Represents the application data and business logic.
    • View: Represents the UI (User Interface) that displays the data.
    • Controller: Handles user input and interaction, processes the request, and returns a view or response.

    Key components of Spring MVC:

    • DispatcherServlet: Acts as the front controller that intercepts all incoming requests and delegates them to appropriate handlers.
    • HandlerMapping: Maps incoming requests to the appropriate controllers.
    • Controller: Handles the requests, processes them, and returns the response.
    • ViewResolver: Resolves the view to render based on the controller’s response.

    2. What is DispatcherServlet in Spring MVC?

    Answer: DispatcherServlet is the front controller in Spring MVC. It is responsible for handling all HTTP requests and responses. It acts as an intermediary between the client and the controllers. DispatcherServlet dispatches the request to appropriate handlers, views, or other components based on configuration.

    Responsibilities of DispatcherServlet:

    • Receives the request and routes it to the appropriate controller.
    • Delegates tasks to HandlerMapping, ViewResolver, and other components.
    • Processes the model and view returned by the controller and renders the final output to the user.

    Configuration in web.xml:

    <servlet>
      <servlet-name>dispatcher</servlet-name>
      <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
      <load-on-startup>1</load-on-startup>
    </servlet>

    <servlet-mapping>
      <servlet-name>dispatcher</servlet-name>
      <url-pattern>/</url-pattern>
    </servlet-mapping>

    3. Explain the flow of a Spring MVC application.

    Answer: The typical flow of a Spring MVC application is as follows:

    1. Client Request: A client sends a request to the server.
    2. DispatcherServlet: The request is intercepted by the DispatcherServlet, which acts as the front controller.
    3. HandlerMapping: DispatcherServlet uses HandlerMapping to determine which controller is responsible for handling the request.
    4. Controller: The request is forwarded to the appropriate controller.
    5. Business Logic: The controller processes the request, possibly interacting with the service and data access layers.
    6. Model: The controller prepares a model that contains the data to be displayed.
    7. ViewResolver: The controller returns a view name to the DispatcherServlet, which uses ViewResolver to resolve the actual view.
    8. Rendering: The view is rendered and the response is sent back to the client.

    4. What is the role of @Controller and @RestController in Spring MVC?

    Answer:

    • @Controller: This annotation is used to define a class as a Spring MVC controller that can handle web requests. A class annotated with @Controller is responsible for returning views (usually JSP or Thymeleaf templates) using ModelAndView or by returning a view name directly.
    @Controller
    public class MyController {
      @RequestMapping("/home")
      public String home() {
        return "home"; // Returns the "home.jsp" or equivalent view
      }
    }
    • @RestController: This is a specialization of @Controller introduced in Spring 4.0. It combines @Controller and @ResponseBody. It is used for RESTful web services and returns data directly in the HTTP response body (e.g., JSON or XML) instead of a view.
    @RestController
    public class MyRestController {
      @RequestMapping("/api/data")
      public ResponseEntity<String> getData() {
        return new ResponseEntity<>("Hello, World!", HttpStatus.OK);
      }
    }

    The key difference is that @RestController is used to create REST APIs, while @Controller is used for traditional MVC applications where views are rendered.

    5. What is @RequestMapping, and how is it used?

    Answer: @RequestMapping is an annotation used to map HTTP requests to handler methods in a controller. It can be applied at the class level and method level.

    • At Class Level: To define a base URL for all methods in the class.
    • At Method Level: To map specific URLs to individual methods.
    @Controller
    @RequestMapping("/user")
    public class UserController {
      @RequestMapping("/profile", @RequestMethod.GET)
      public String getUserProfile(Model model) {
        // Logic for retrieving user profile
        model.addAttribute("user", new User());
        return "profile"; // Returns profile.jsp
      }
    }
    • value: The URL to be mapped.
    • method: The HTTP method (GET, POST, PUT, DELETE) for which this mapping applies.

    6. What is @GetMapping and @PostMapping, and how do they differ from @RequestMapping?

    Answer:

    • @GetMapping: A shortcut for @RequestMapping(method = RequestMethod.GET) used for mapping GET requests.
    • @PostMapping: A shortcut for @RequestMapping(method = RequestMethod.POST) used for mapping POST requests.
    @GetMapping("/users")
    public String getAllUsers() {
      // Handle GET request
      return "userList";
    }

    @PostMapping("/users")
    public String createUser(@ModelAttribute User user) {
      // Handle POST request
      return "userDetail";
    }

    These annotations are more readable and specific compared to @RequestMapping, which can handle multiple HTTP methods.

    7. What is a ModelAndView in Spring MVC?

    Answer: ModelAndView is a class in Spring MVC used to encapsulate both the model (data) and the view (UI template) in a single return value from a controller method. It helps in passing data from the controller to the view.

    @Controller
    public class MyController {
      @RequestMapping("/home")
      public ModelAndView home() {
        ModelAndView mav = new ModelAndView("home"); // View name
        mav.addObject("message", "Welcome to Spring MVC!"); // Add data to the model
        return mav;
      }
    }

    Here, ModelAndView holds the view name (home) and the model data (message).

    8. What is the role of @ModelAttribute in Spring MVC?

    Answer: @ModelAttribute is used to bind a method parameter or method return value to a named model attribute. It allows data to be passed between the controller and the view.

    Usage:

    • To bind form data to a model object.
    • To prepopulate a model object before a request is handled.
    @Controller
    public class UserController {
      @PostMapping("/saveUser")
      public String saveUser(@ModelAttribute User user) {
        // Process the user object from the form
        return "userDetail";
      }
    }
    @ModelAttribute("user")
    public User getUser() {
        return new User(); // Prepopulate user object
    }

    The @ModelAttribute annotation binds form fields or request parameters to the model attribute and makes it available in the view.

    9. What is @RequestParam, and how does it work?

    Answer: @RequestParam is used to extract query parameters or form data from an HTTP request and bind them to method parameters.

    @GetMapping("/search")
    public String search(@RequestParam("keyword") String keyword, Model model) {
      // Use the 'keyword' query parameter for search
      model.addAttribute("result", searchService.search(keyword));
      return "searchResults";
    }

    @RequestParam(value = "paramName", required = false, defaultValue = "default") can be used to make the request parameter optional or to provide a default value if it’s not present in the request.

    10. What is @ModelAttribute used for in Spring MVC, and how does it differ from @RequestParam?

    Answer: @ModelAttribute is used to bind request parameters to a model object or to expose a model attribute to the view.

    • Binding Request Parameters: It can be used to bind form data or request parameters to a model object. When used on a method parameter, it creates or binds an instance of the model class with request parameters.
    • Exposing Model Attributes: It can also be used on a method to add attributes to the model, which can then be used in the view.
    @PostMapping("/submitForm")
    public String submitForm(@ModelAttribute User user) {
      // Process the user object
      return "result";
    }
    @Controller
    public class MyController {
      @ModelAttribute("welcomeMessage")
      public String welcomeMessage() {
        return "Welcome to the application!";
      }
    }

    Here, @ModelAttribute provides more flexibility than @RequestParam, allowing complex data binding and prepopulating model attributes.

    11. What is the difference between @ControllerAdvice and @ExceptionHandler?

    Answer:

    • @ControllerAdvice: A specialized form of @Component that provides global exception handling and model enhancements across multiple controllers. It allows you to handle exceptions, add model attributes, and apply global configurations across the application. Example:
    @ControllerAdvice
    public class GlobalExceptionHandler {
      @ExceptionHandler(Exception.class)
      public ModelAndView handleException(Exception ex) {
        ModelAndView mav = new ModelAndView("error"); // View name
        mav.addObject("errorMessage", ex.getMessage()); // Add data to the model
        return mav;
      }
    }

    @ExceptionHandler: Used within a specific controller or @ControllerAdvice class to handle exceptions thrown by controller methods. It is more specific and local to the controller or advice class where it is defined.

    12. What is @InitBinder, and how is it used?

    Answer: @InitBinder is used to customize the binding of web request parameters to method parameters in controllers. It allows you to add custom editors or converters to handle complex types or data transformations.

    • Example:
    @Controller
    public class MyController {
      @InitBinder
      public void initBinder(WebDataBinder binder) {
        binder.registerCustomEditor(Date.class, new CustomDateEditor(new SimpleDateFormat("yyyy-MM-dd"), false));
      }

      @PostMapping("/date")
      public String handleDate(@RequestParam Date date) {
        // Handle the date parameter
        return "result";
      }
    }

    In this example, @InitBinder is used to register a custom date editor to handle date formatting.

    13. What is @ResponseStatus, and how is it used in a custom exception class?

    Answer: @ResponseStatus is used to specify the HTTP status code that should be returned for a specific exception. It can be applied to exception classes to define a default response status when the exception is thrown.

    • Example:
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public class InvalidInputException extends RuntimeException {
      public InvalidInputException(String message) {
        super(message);
      }
    }

    In this example, whenever InvalidInputException is thrown, the HTTP response will have a status code of 400 Bad Request.

    14. What are @CrossOrigin annotations, and how do you use them?

    Answer: @CrossOrigin is used to handle Cross-Origin Resource Sharing (CORS) in Spring MVC applications. It allows you to specify which origins are allowed to access your resources, enabling cross-origin requests.

    • Example:
    @RestController
    @CrossOrigin("http://example.com")
    public class MyRestController {
      @GetMapping("/data")
      public ResponseEntity<String> getData() {
        return new ResponseEntity<>("Data from server", HttpStatus.OK);
      }
    }

    This configuration allows requests from http://example.com to access the /data endpoint.

    15. What is @SessionAttributes, and how does it work?

    Answer: @SessionAttributes is used to store model attributes in the HTTP session. It allows you to keep model attributes across multiple requests from the same user, making them available during the entire session.

    • Example:
    @Controller
    @SessionAttributes("user")
    public class UserController {
      @ModelAttribute("user")
      public User getUser() {
        return new User(); // Prepopulate user object
      }

      @PostMapping("/updateUser")
      public String updateUser(@ModelAttribute User user) {
        // Update user information
        return "userDetail";
      }
    }

    In this example, the user attribute will be stored in the session and be available across multiple requests.

    16. What is @RequestHeader, and how do you use it?

    Answer: @RequestHeader is used to bind HTTP request headers to method parameters in a controller. It allows you to access header values directly within controller methods.

    • Example:
    @GetMapping("/headers")
    public String getHeaders(@RequestHeader("User-Agent") String userAgent) {
      // Process the User-Agent header value
      return "headers";
    }

    In this example, the User-Agent header is accessed and used in the method.

    17. How can you implement file upload functionality in Spring MVC?

    Answer: To handle file uploads in Spring MVC, you can use MultipartFile to represent uploaded files. You also need to configure multipart file handling in your Spring application.

    • Configuration:
    <multipart-config>
      <max-file-size>100MB</max-file-size>
      <max-request-size>100MB</max-request-size>
    </multipart-config>
    • Controller:
    @Controller
    public class FileUploadController {
      @PostMapping("/upload")
      public String handleFileUpload(@RequestParam("file") MultipartFile file) {
        if (!file.isEmpty()) {
          // Process the file
          String fileName = file.getOriginalFilename();
          // Save the file or process it
        }
        return "uploadResult";
      }
    }

    In this example, the uploaded file is accessed via MultipartFile, and you can process it as needed.

    18. What is the @RequestMapping method parameter params, and how is it used?

    Answer: The @RequestMapping method parameter params is used to specify that a request must contain specific query parameters for the handler method to be invoked. It is useful for request parameter filtering.

    • Example:
    @GetMapping("/users", params = "active=true")
    public String getActiveUsers() {
      // Handle request with "active=true" parameter
      return "activeUsers";
    }

    In this example, the handler method will only be invoked if the request contains the parameter active=true.

    19. What are ViewResolvers in Spring MVC, and how do they work?

    Answer: ViewResolvers are used to resolve view names into actual view objects. They determine which view should be rendered based on the view name returned by a controller.

    Common Types of ViewResolvers:

    • InternalResourceViewResolver: Resolves view names to JSP files or other server-side technologies.
    • ThymeleafViewResolver: Resolves view names to Thymeleaf templates.
    • XmlViewResolver: Resolves view names based on XML configuration.
    • Example:
    @Bean
    public ViewResolver viewResolver() {
      InternalResourceViewResolver resolver = new InternalResourceViewResolver();
      resolver.setPrefix("/WEB-INF/views/");
      resolver.setSuffix(".jsp");
      return resolver;
    }

    Here, InternalResourceViewResolver is configured to resolve view names to JSP files located in the /WEB-INF/views/ directory.

    20. What is @RequestMapping in Spring MVC, and how does it differ from @GetMapping, @PostMapping, etc.?

    Answer: @RequestMapping is a general-purpose annotation for mapping HTTP requests to handler methods. It can be used with various HTTP methods by specifying the method attribute.

    • @RequestMapping: General-purpose and can handle all HTTP methods.
    • @GetMapping: Shortcut for @RequestMapping(method = RequestMethod.GET), specifically for GET requests.
    • @PostMapping: Shortcut for @RequestMapping(method = RequestMethod.POST), specifically for POST requests.
    • @PutMapping: Shortcut for @RequestMapping(method = RequestMethod.PUT), specifically for PUT requests.
    • @DeleteMapping: Shortcut for @RequestMapping(method = RequestMethod.DELETE), specifically for DELETE requests.
    • Example:
    @RequestMapping(value = "/process", method = RequestMethod.POST)
    public String processForm(@RequestParam("name") String name) {
      // Handle POST request
      return "result";
    }

    @GetMapping("/greet")
    public String greetUser() {
      // Handle GET request
      return "greet";
    }

    21. How does Spring MVC handle form submissions?

    Answer: Spring MVC handles form submissions by binding form data to model objects and processing it in controller methods. The process involves:

    • Form Tag Library: Using the <form:form> tag from the Spring Form Tag Library to create HTML forms.
    • Model Binding: Automatically binding form fields to model attributes using @ModelAttribute.
    • Form Validation: Validating form data using JSR-303/JSR-380 (Bean Validation) annotations and the @Valid annotation.

    Example:

    <!-- form.jsp -->
    <form:form action="/submit" method="post" modelAttribute="user">
      <form:input path="username" />
      <form:input path="email" />
      <input type="submit" value="Submit" />
    </form:form>

    Controller:

    @Controller
    public class FormController {
      @PostMapping("/submit")
      public String submitForm(@ModelAttribute @Valid User user, BindingResult result) {
        if (result.hasErrors()) {
          return "form";
        }
        // Process the valid user object
        return "result";
      }
    }

    22. What is the purpose of @PathVariable in Spring MVC, and how do you use it?

    Answer: @PathVariable is used to extract values from URI template variables and bind them to method parameters. It is useful for extracting data from the URL path.

    Example:

    @Controller
    public class ProductController {
      @GetMapping("/products/{productId}")
      public String getProduct(@PathVariable("productId") Long productId, Model model) {
        Product product = productService.findById(productId);
        model.addAttribute("product", product);
        return "productDetail";
      }
    }

    24. How do you configure Spring MVC to use custom error pages?

    Answer: Custom error pages can be configured in Spring MVC using the ErrorController interface or by defining error views in the web.xml or using configuration classes.

    Using ErrorController:

    @Controller
    public class CustomErrorController implements ErrorController {
      @RequestMapping("/error")
      public String handleError() {
        // Return custom error view
        return "errorPage";
      }

      @Override
      public String getErrorPath() {
        return "/error";
      }
    }

    Configuring in web.xml:

    <error-page>
      <error-code>404</error-code>
      <location>/WEB-INF/views/404.jsp</location>
    </error-page>

    Using Configuration Class:

    @Configuration
    public class WebConfig implements WebMvcConfigurer {
      @Bean
      public ErrorViewResolver errorViewResolver() {
        return new MyErrorViewResolver();
      }
    }

    25. What is @RequestPart and how is it used?

    Answer: @RequestPart is used to handle multipart file uploads where the file is part of the request payload. It allows you to access parts of the request, such as files and JSON, separately.

    Example:

    @PostMapping("/upload")
    public String handleFileUpload(@RequestPart("file") MultipartFile file,
        @RequestPart("metadata") String metadata) {

    26. How do you handle file uploads with a specific size limit in Spring MVC?

    Answer: To handle file uploads with size limits, you can configure multipart file upload settings in application.properties or application.yml.

    In application.properties:

    spring.servlet.multipart.max-file-size=10MB
    spring.servlet.multipart.max-request-size=10MB

    In application.yml:

    spring:
      servlet:
        multipart:
          max-file-size: 10MB
          max-request-size: 10MB

    You can also configure file upload settings in a MultipartConfigElement bean if you need custom configurations.

    Example:

    @Bean
    public MultipartConfigElement multipartConfigElement() {
      MultipartConfigFactory factory = new MultipartConfigFactory();
        factory.setMaxFileSize(DataSize.ofMegabytes(10));
        factory.setMaxRequestSize(DataSize.ofMegabytes(10));
        return factory.createMultipartConfig();
      }

    27. How can you configure Spring MVC to use different view technologies, like Thymeleaf or Freemarker?

    Answer: Spring MVC can be configured to use different view technologies by setting up the appropriate ViewResolver beans in your configuration.

    For Thymeleaf:

    @Configuration
    public class ThymeleafConfig {
      @Bean
      public SpringTemplateEngine templateEngine() {
        SpringTemplateEngine templateEngine = new SpringTemplateEngine();
        // Configure additional properties if needed
        return templateEngine;
      }

      @Bean
      public ThymeleafViewResolver thymeleafViewResolver() {
        ThymeleafViewResolver viewResolver = new ThymeleafViewResolver();
        viewResolver.setTemplateEngine(templateEngine);
        viewResolver.setPrefix("/WEB-INF/templates/");
        viewResolver.setSuffix(".html");
        return viewResolver;
      }
    }

    For Freemarker:

    @Configuration
    public class FreemarkerConfig {
      @Bean
      public FreeMarkerConfigurer freemarkerConfigurer() {
        FreeMarkerConfigurer configurer = new FreeMarkerConfigurer();
        configurer.setTemplateLoaderPath("/WEB-INF/freemarker/");
        return configurer;
      }

      @Bean
      public FreeMarkerViewResolver freemarkerViewResolver() {
        FreeMarkerViewResolver resolver = new FreeMarkerViewResolver();
        resolver.setPrefix("");
        resolver.setSuffix(".ftl");
        resolver.setCache(true);
        resolver.setOrder(0);
        return resolver;
      }
    }

    28. How do you perform internationalization (i18n) in a Spring MVC application?

    Answer: Internationalization in Spring MVC can be performed by using message resource bundles and configuring LocaleResolver and MessageSource.

    Configuration:

    @Configuration
    public class WebConfig implements WebMvcConfigurer {
      @Bean
      public LocaleResolver localeResolver() {
        SessionLocaleResolver localeResolver = new SessionLocaleResolver();
        localeResolver.setDefaultLocale(Locale.ENGLISH);
        return localeResolver;
      }

      @Bean
      public MessageSource messageSource() {
        ResourceBundleMessageSource messageSource = new ResourceBundleMessageSource();
        messageSource.setBasename("messages");
        messageSource.setDefaultEncoding("UTF-8");
        return messageSource;
      }
    }

    Messages Property File:

    • messages.properties (default)
    • messages_fr.properties (French translations)

    Usage in JSP:

    <spring:message code="welcome.message" />

    In this example, LocaleResolver determines the user’s locale, and MessageSource provides localized messages based on the locale.
