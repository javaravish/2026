    Spring JPA Interview Questions

    1. What is Spring Data JPA?

    Answer: Spring Data JPA is part of the Spring Data project that simplifies database access using JPA (Java Persistence API). It provides a repository abstraction layer to facilitate CRUD operations and querying. It integrates with JPA providers like Hibernate and simplifies data access by reducing boilerplate code.

    Example:

    @Repository
    public interface UserRepository extends JpaRepository<User, Long> {
      // Custom query methods can be defined here
      User findByUsername(String username);
    }

    2. What is the role of @Entity in Spring Data JPA?

    Answer: The @Entity annotation is used to mark a class as an entity, which means it represents a table in the database. Each instance of the class corresponds to a row in the table.

    Example:

    @Entity
    @Table(name = "users")
    public class User {
      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      private String username;
      private String email;

      // Getters and setters
    }

    3. How does Spring Data JPA support custom queries?

    Answer: Spring Data JPA allows custom queries using the @Query annotation on repository methods. You can define JPQL (Java Persistence Query Language) or native SQL queries to perform complex data retrieval operations.

    Example:

    @Repository
    public interface UserRepository extends JpaRepository<User, Long> {

      @Query("SELECT u FROM User u WHERE u.username = :username")
      User findByUsername(@Param("username") String username);

      @Query(value = "SELECT * FROM users WHERE email = ?1", nativeQuery = true)
      User findByEmail(String email);
    }

    4. What is the purpose of the @Id annotation?

    Answer: The @Id annotation marks a field as the primary key of the entity. Each entity must have a field annotated with @Id, which uniquely identifies each row in the corresponding table.

    Example:

    @Entity
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      private String username;
      private String email;

      // Getters and setters
    }

    5. What are @ManyToOne and @OneToMany associations?

    Answer:

    • @ManyToOne: Defines a many-to-one relationship between entities. Many instances of the source entity can be associated with a single instance of the target entity.
    • @OneToMany: Defines a one-to-many relationship between entities. One instance of the source entity can be associated with multiple instances of the target entity.

    Example:

    @Entity
    public class Order {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @ManyToOne
      @JoinColumn(name = "user_id")
      private User user;

      // Other fields
    }
    @Entity
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @OneToMany(mappedBy = "user")
      private List<Order> orders;

      // Other fields
    }

    6. What is the @Transactional annotation used for in Spring JPA?

    Answer: The @Transactional annotation is used to manage transactions in Spring JPA. It ensures that a series of operations within a method are executed in a single transaction context. It handles transaction boundaries, rollbacks, and commits automatically.

    Example:

    @Service
    public class UserService {

      @Autowired
      private UserRepository userRepository;

      @Transactional
      public void createUser(User user) {
        userRepository.save(user);
        // Other operations that should be part of the transaction
      }
    }

    7. What is the EntityManager and how is it used in Spring JPA?

    Answer: EntityManager is the primary JPA interface used for interacting with the persistence context. It manages the lifecycle of entities, performs CRUD operations, and queries the database.

    Example:

    @Service
    public class UserService {

      @PersistenceContext
      private EntityManager entityManager;

      public User findUserById(Long id) {
        return entityManager.find(User.class, id);
      }

      public void persistUser(User user) {
        entityManager.persist(user);
      }
    }

    8. What is the @Query annotation used for in Spring Data JPA?

    Answer: The @Query annotation is used to define custom JPQL or SQL queries in repository methods. It allows you to execute complex queries directly from the repository layer.

    Example:

    @Repository
    public interface UserRepository extends JpaRepository<User, Long> {

      @Query("SELECT u FROM User u WHERE u.email LIKE %:email%")
      List<User> findByEmailLike(@Param("email") String email);
    }

    9. What is the difference between @OneToOne and @ManyToMany associations?

    Answer:

    • @OneToOne: Defines a one-to-one relationship where each instance of the source entity is associated with exactly one instance of the target entity and vice versa.
    • @ManyToMany: Defines a many-to-many relationship where multiple instances of the source entity can be associated with multiple instances of the target entity and vice versa.

    Example:

    @Entity
    public class Employee {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @OneToOne
      @JoinColumn(name = "address_id")
      private Address address;

      // Other fields
    }
    @Entity
    public class Project {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @ManyToMany
      @JoinTable(
        name = "employee_project",
        @JoinColumns = @JoinColumn(name = "employee_id"),
        @InverseJoinColumns = @JoinColumn(name = "project_id")
      )
      private Set<Employee> employees;

      // Other fields
    }

    10. How does Spring Data JPA handle lazy loading and eager loading?

    Answer:

    • Lazy Loading: Entities or collections are fetched on demand. It uses proxy objects to defer the loading of related entities until they are accessed.
    • Eager Loading: Entities or collections are fetched immediately with the parent entity. It retrieves all related data in a single query.

    Example:

    @Entity
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @OneToMany(fetch = FetchType.LAZY, mappedBy = "user")
      private List<Order> orders;

      // Other fields
    }
    @Entity
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @OneToMany(fetch = FetchType.EAGER, mappedBy = "user")
      private List<Order> orders;

      // Other fields
    }

    11. What is the purpose of the @Table annotation in Spring Data JPA?

    Answer: The @Table annotation is used to specify the name of the database table that the entity will map to. It allows you to define table attributes like the name, schema, and catalog.

    Example:

    @Entity
    @Table(name = "users")
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      private String username;
      private String email;

      // Getters and setters
    }

    12. What are @Embedded and @Embeddable annotations?

    Answer:

    • @Embeddable: Marks a class as an embeddable type. This class can be embedded in an entity.
    • @Embedded: Used within an entity to include an embeddable type.

    Example:

    @Embeddable
    public class Address {

      private String street;
      private String city;
      private String state;
      private String zipCode;

      // Getters and setters
    }
    @Entity
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      private String username;

      @Embedded
      private Address address;

      // Getters and setters
    }

    13. How do you define a composite primary key in Spring Data JPA?

    Answer: To define a composite primary key, use the @Embeddable annotation for the primary key class and @EmbeddedId in the entity class.

    Example:

    @Embeddable
    public class OrderId implements Serializable {

      private Long orderId;
      private Long productId;

      // Getters, setters, equals, and hashCode methods
    }
    @Entity
    public class Order {

      @EmbeddedId
      private OrderId id;

      private int quantity;

      // Other fields
    }

    14. What are @Query, @Modifying, and @Transactional annotations used for in Spring Data JPA?

    Answer:

    • @Query: Used to define custom JPQL or SQL queries directly on repository methods.
    • @Modifying: Indicates that the query is a modifying query (e.g., UPDATE, DELETE) and should be executed within a transaction.
    • @Transactional: Manages transaction boundaries, ensuring that modifying queries are executed within a transactional context.

    Example:

    @Repository
    public interface UserRepository extends JpaRepository<User, Long> {

      @Modifying
      @Transactional
      @Query("UPDATE User u SET u.email = :email WHERE u.username = :username")
      void updateEmailByUsername(@Param("username") String username, @Param("email") String email);
    }

    15. What is EntityGraph and how is it used in Spring Data JPA?

    Answer: EntityGraph allows you to define fetch plans for JPA queries, specifying which associations should be eagerly loaded. It helps optimize performance by controlling the fetch strategy.

    Example:

    @Entity
    @NamedEntityGraph(name = "User.detail",
      @NamedAttributeNode("orders"))
    public class User {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @OneToMany(fetch = FetchType.LAZY, mappedBy = "user")
      private List<Order> orders;

      // Other fields
    }
    @Repository
    public interface UserRepository extends JpaRepository<User, Long> {

      @EntityGraph(value = "User.detail", type = EntityGraph.EntityGraphType.LOAD)
      User findById(Long id);
    }

    16. What is Criteria API and how is it used in Spring Data JPA?

    Answer: The Criteria API is a JPA feature for creating dynamic and type-safe queries programmatically. It allows building queries using a fluent API instead of writing JPQL or SQL.

    Example:

    @Service
    public class UserService {

      @Autowired
      private EntityManager entityManager;

      public List<User> findUsersByUsername(String username) {

        CriteriaBuilder cb = entityManager.getCriteriaBuilder();
        CriteriaQuery<User> cq = cb.createQuery(User.class);
        Root<User> user = cq.from(User.class);
        cq.select(user).where(cb.equal(user.get("username"), username));

        TypedQuery<User> query = entityManager.createQuery(cq);
        return query.getResultList();

      }
    }
