    1. What are the different ways to pass data to a controller in Spring MVC?

    Answer: Data can be passed to a Spring MVC controller in several ways:

    • Query Parameters: Use @RequestParam to extract query parameters from the URL.
    @GetMapping("/greet")
    public String greet(@RequestParam("name") String name, Model model) {
      model.addAttribute("message", "Hello, " + name);
      return "greet";
    }
    • Path Variables: Use @PathVariable to capture values from the URI path.
    @GetMapping("/user/{id}")
    public String getUser(@PathVariable("id") Long userId, Model model) {
      // Fetch user details using userId
      return "user";
    }
    • Form Data: Use @ModelAttribute to bind form data to a model object.
    @PostMapping("/submit")
    public String submitForm(@ModelAttribute User user) {
      // Process the user object
      return "result";
    }
    • Request Body: Use @RequestBody to bind the request body to a model object, typically for JSON or XML payloads.
    @PostMapping("/create")
    public ResponseEntity<User> createUser(@RequestBody User user) {
      // Create user logic
      return new ResponseEntity<>(user, HttpStatus.CREATED);
    }

    2. What are the different types of responses that a Spring MVC controller can return?

    Answer:

    • View Names: Return the name of a view (e.g., JSP, Thymeleaf) which will be resolved by a view resolver.
    @GetMapping("/home")
    public String home() {
      return "home"; // Returns view named 'home'
    }
    • ModelAndView: Return a ModelAndView object that contains both the view name and model attributes.
    @GetMapping("/profile")
    public ModelAndView profile() {
      ModelAndView mav = new ModelAndView("profile");
      mav.addObject("user", new User());
      return mav;
    }
    • ResponseEntity: Return a ResponseEntity object that allows setting the HTTP status and response body.
    @GetMapping("/data")
    public ResponseEntity<Map<String, String>> getData() {
      Map<String, String> response = new HashMap<>();
      response.put("key", "value");
      return new ResponseEntity<>(response, HttpStatus.OK);
    }
    • String: Return a direct response body, often used with @ResponseBody.
    @GetMapping("/message")
    @ResponseBody
    public String getMessage() {
      return "Hello, World!";
    }
    • Object: Return an object that will be automatically converted to JSON or XML if @ResponseBody is used.
    @GetMapping("/object")
    @ResponseBody
    public User getUser() {
      return new User("John", "Doe");
    }

    3. How do you handle file uploads in Spring MVC?

    Answer: File uploads can be handled using the MultipartFile class. You need to configure a multipart resolver and use MultipartFile as a parameter in your controller method.

    @PostMapping("/upload")
    public String handleFileUpload(@RequestParam("file") MultipartFile file, Model model) {
      if (!file.isEmpty()) {
        String fileName = file.getOriginalFilename();
        // Save the file or process it
        model.addAttribute("fileName", fileName);
      }
      return "uploadResult";
    }

    }

    5. What is the purpose of @ResponseBody annotation in a controller method?

    Answer: The @ResponseBody annotation indicates that the return value of the method should be written directly to the HTTP response body, bypassing view resolution. This is commonly used to return JSON or XML data.

    @GetMapping("/json")
    @ResponseBody
    public User getUser() {
      return new User("John", "Doe"); // Returned as JSON if Jackson is on the classpath
    }

    6. How do you handle HTTP headers in Spring MVC controllers?

    Answer: You can handle HTTP headers by using the @RequestHeader annotation to extract header values from the request and by setting headers in the ResponseEntity object.

    @GetMapping("/headers")
    public String handleHeaders(@RequestHeader("User-Agent") String userAgent, Model model) {
      model.addAttribute("userAgent", userAgent);
      return "headers";
    }

    @PostMapping("/response")
    public ResponseEntity<String> handleResponse() {
      HttpHeaders headers = new HttpHeaders();
      headers.add("Custom-Header", "value");
      return new ResponseEntity<>("Response with custom header", headers, HttpStatus.OK);
    }

    7. What is the difference between @RequestParam and @PathVariable annotations?

    Answer:

    • @RequestParam: Used to extract query parameters from the URL or form parameters from a POST request. It is typically used for optional or multiple parameters.
    @GetMapping("/search")
    public String search(@RequestParam("query") String query) {
      // Handle query parameter
      return "searchResults";
    }
    • @PathVariable: Used to extract values from the URI path. It is typically used for mandatory parameters that are part of the URL path.
    @GetMapping("/user/{id}")
    public String getUser(@PathVariable("id") Long id) {
      // Handle path variable
      return "userProfile";
    }

    8. How do you handle different content types in Spring MVC?

    Answer: You can handle different content types by using @RequestMapping with the consumes and produces attributes to specify the media types for request and response.

    @PostMapping(value = "/submit", consumes = "application/json", produces = "application/json")
    @ResponseBody
    public ResponseEntity<User> submitUser(@RequestBody User user) {
      // Process the user and return as JSON
      return new ResponseEntity<>(user, HttpStatus.CREATED);
    }

    @GetMapping(value = "/data", produces = "application/xml")
    @ResponseBody
    public User getUser() {
      // Return XML response
      return new User("John", "Doe");
    }

    9. How can you handle redirect and forward in Spring MVC?

    Answer:

    • Redirect: Use the redirect: prefix in the return value of a controller method to redirect to another URL.
    @PostMapping("/submit")
    public String submit() {
      return "redirect:/success";
    }
    • Forward: Use the forward: prefix in the return value to forward the request to another URL within the application.
    @GetMapping("/home")
    public String home() {
      return "forward:/dashboard";
    }

    11. How can you validate request parameters and handle validation errors in Spring MVC?

    Answer: Request parameters can be validated using annotations like @Valid or @Validated on model objects. Validation errors can be handled using BindingResult or @ExceptionHandler.

    @PostMapping("/register")
    public String registerUser(@Valid @ModelAttribute User user, BindingResult result) {
      if (result.hasErrors()) {
        return "registrationForm";
      }
      return "registrationSuccess";
    }

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(MethodArgumentNotValidException.class)
      public ResponseEntity<Map<String, String>> handleValidationExceptions(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getAllErrors().forEach(error -> {
          errors.put("error", error.getDefaultMessage());
        });
        return new ResponseEntity<>(errors, HttpStatus.BAD_REQUEST);
      }
    }

    13. What is the difference between @Component, @Service, @Repository, and @Controller annotations?

    Answer:

    • @Component: A generic stereotype for any Spring-managed component.
    • @Service: Specialized stereotype for service layer components, generally used for business logic.
    • @Repository: Specialized stereotype for persistence layer components, which also provides additional functionality like exception translation.
    • @Controller: Specialized stereotype for web layer components, used for defining controllers in Spring MVC.

    14. How can you use custom validators in Spring MVC?

    Answer: Custom validators can be created by implementing the Validator interface and annotating your model class with @Valid or @Validated. The custom validator is then registered with Spring.

    @Component
    public class UserValidator implements Validator {

      @Override
      public boolean supports(Class<?> clazz) {
        return User.class.isAssignableFrom(clazz);
      }

      @Override
      public void validate(Object target, Errors errors) {
        User user = (User) target;
        if (user.getName() == null) {
          errors.rejectValue("name", "field.required");
        }
      }
    }

    Spring Exception Handling Interview Questions and Answers

    1. What is the role of @ExceptionHandler in Spring MVC?

    Answer: The @ExceptionHandler annotation in Spring MVC is used to handle exceptions thrown by handler methods in a controller. It allows you to define methods in your controller that will be invoked when a specific exception is thrown, enabling you to handle exceptions gracefully and return appropriate responses.

    Example:

    @Controller
    public class MyController {

      @RequestMapping("/example")
      public String example() {
        throw new RuntimeException("Example exception");
      }

      @ExceptionHandler(RuntimeException.class)
      public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("An error occurred: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
      }
    }

    2. How does @ControllerAdvice work in Spring MVC?

    Answer: @ControllerAdvice is a specialization of the @Component annotation that allows you to define global exception handling across multiple controllers. It can be used to handle exceptions, bind data across controllers, and apply global model attributes.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(RuntimeException.class)
      public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("Global error: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
      }
    }

    3. What is the difference between @ExceptionHandler and @ControllerAdvice?

    Answer:

    • @ExceptionHandler: This annotation is used at the controller level to handle exceptions thrown by that specific controller. It allows you to define exception handling methods within the controller itself.
    • @ControllerAdvice: This annotation is used at the global level to handle exceptions across multiple controllers. It allows you to define exception handling methods that apply to all controllers within the application.

    Example:

    @Controller
    public class MyController {
      // Controller-specific exception handling
    }

    @ControllerAdvice
    public class GlobalExceptionHandler {
      // Global exception handling
    }

    4. How do you handle custom exceptions in Spring MVC?

    Answer: To handle custom exceptions in Spring MVC, you can define your own exception classes and then use @ExceptionHandler or @ControllerAdvice to handle these exceptions.

    Example:

    // Custom Exception
    public class CustomException extends RuntimeException {
      public CustomException(String message) {
        super(message);
      }
    }

    @Controller
    public class MyController {

      @RequestMapping("/custom")
      public String custom() {
        throw new CustomException("Custom exception occurred");
      }

      @ExceptionHandler(CustomException.class)
      public ResponseEntity<String> handleCustomException(CustomException ex) {
        return new ResponseEntity<>("Custom error: " + ex.getMessage(), HttpStatus.BAD_REQUEST);
      }
    }

    5. How can you return a JSON response from an exception handler method?

    Answer: You can return a JSON response from an exception handler method by using the @ResponseBody annotation along with @ExceptionHandler. This ensures that the response body is serialized to JSON format.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(RuntimeException.class)
      @ResponseBody
      public ResponseEntity<Map<String, String>> handleRuntimeException(RuntimeException ex) {
        Map<String, String> response = new HashMap<>();
        response.put("error", ex.getMessage());
        return new ResponseEntity<>(response, HttpStatus.INTERNAL_SERVER_ERROR);
      }
    }

    6. What is the use of @ResponseStatus annotation?

    Answer: The @ResponseStatus annotation is used to specify the HTTP status code that should be returned when a specific exception is thrown. It can be applied to exception classes or exception handler methods to set the desired HTTP status.

    Example:

    @ResponseStatus(HttpStatus.NOT_FOUND)
    public class ResourceNotFoundException extends RuntimeException {
      public ResourceNotFoundException(String message) {
        super(message);
      }
    }

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(ResourceNotFoundException.class)
      public ResponseEntity<String> handleResourceNotFoundException(ResourceNotFoundException ex) {
        return new ResponseEntity<>(ex.getMessage(), HttpStatus.NOT_FOUND);
      }
    }

    7. How can you log exceptions in a global exception handler?

    Answer: You can log exceptions in a global exception handler using a logging framework such as SLF4J or Log4j. By injecting a logger into your @ControllerAdvice class, you can log the details of the exceptions before returning a response.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      private static final Logger logger = LoggerFactory.getLogger(GlobalExceptionHandler.class);

      @ExceptionHandler(Exception.class)
      public ResponseEntity<String> handleException(Exception ex) {
        logger.error("Exception occurred: ", ex);
        return new ResponseEntity<>("An error occurred: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
      }
    }

    8. How do you handle validation errors in Spring MVC?

    Answer: Validation errors in Spring MVC can be handled using @Valid or @Validated annotations on method parameters in controllers, along with binding result objects to capture validation errors.

    Example:

    @Controller
    public class MyController {

      @PostMapping("/submit")
      public String submitForm(@Valid @ModelAttribute("form") MyForm form, BindingResult result) {
        if (result.hasErrors()) {
          return "form";
        }
        return "success";
      }
    }

    To handle validation errors globally:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(MethodArgumentNotValidException.class)
      @ResponseBody
      public ResponseEntity<Map<String, String>> handleValidationExceptions(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getAllErrors().forEach(error -> {
          String fieldName = (FieldError) error.getField();
          String errorMessage = error.getDefaultMessage();
          errors.put(fieldName, errorMessage);
        });
        return new ResponseEntity<>(errors, HttpStatus.BAD_REQUEST);
      }
    }

    9. What is the @ResponseBody annotation used for in exception handling?

    Answer: The @ResponseBody annotation indicates that the return value of a method should be used as the response body of the HTTP response. When used in exception handling, it ensures that the exception details or custom error messages are serialized into the response body, typically in JSON format.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(IllegalArgumentException.class)
      @ResponseBody
      public Map<String, String> handleIllegalArgumentException(IllegalArgumentException ex) {
        Map<String, String> response = new HashMap<>();
        response.put("error", ex.getMessage());
        return response;
      }
    }

    10. How do you handle HTTP status codes in exception handling?

    Answer: HTTP status codes can be handled in exception handling by returning appropriate ResponseEntity objects with the desired status code. You can set the status code directly in the ResponseEntity or use the @ResponseStatus annotation on custom exceptions.

    Example:

    @ControllerAdvice
    public class GlobalExceptionHandler {

      @ExceptionHandler(IllegalArgumentException.class)
      public ResponseEntity<String> handleIllegalArgumentException(IllegalArgumentException ex) {
        return new ResponseEntity<>("Invalid argument: " + ex.getMessage(), HttpStatus.BAD_REQUEST);
      }

      @ExceptionHandler(ResourceNotFoundException.class)
      public ResponseEntity<String> handleResourceNotFoundException(ResourceNotFoundException ex) {
        return new ResponseEntity<>("Resource not found: " + ex.getMessage(), HttpStatus.NOT_FOUND);
      }
    }
