    Spring Interview Questions

    1. What is Spring Framework?

    Answer: Spring is an open-source framework primarily used for enterprise Java development. Its core features include Dependency Injection (DI), Aspect-Oriented Programming (AOP), and simplified handling of database transactions. It provides support for building scalable, loosely coupled applications.

    2. What is Inversion of Control (IoC) in Spring?

    Answer: Inversion of Control (IoC) refers to the concept where the control of object creation and dependency management is transferred from the application code to the Spring container. Instead of the application managing the lifecycle and dependencies of objects, the Spring framework does this based on the configuration provided via XML, annotations, or Java code.

    3. What is Dependency Injection in Spring?

    Answer: Dependency Injection (DI) is a design pattern used by the Spring IoC container to resolve dependencies between objects. DI can be performed in two ways:

    • Constructor Injection: Dependencies are injected through a class constructor.
    • Setter Injection: Dependencies are injected via setter methods.

    Example:

    @Autowired
    private MyService myService;

    4. What are the benefits of using Dependency Injection (DI) in Spring?

    Answer:

    • Loose coupling: Objects do not need to create or manage their dependencies. The Spring container does this, resulting in more flexible, reusable, and maintainable code.
    • Easier testing: Since dependencies are injected, it becomes easier to use mock objects or other implementations during unit testing.
    • Better configuration management: Dependencies are centralized in configuration files or annotations.
    • Easier integration: DI allows easy integration of third-party libraries without modifying business logic.

    5. Explain the Spring IoC container. What are its responsibilities?

    Answer: The Spring IoC container is responsible for:

    • Creating and managing the lifecycle of Spring beans.
    • Resolving and injecting dependencies (DI).
    • Managing bean scopes (singleton, prototype, etc.).
    • Handling configuration metadata (annotations, XML, Java configuration).
    • Wiring beans together based on configurations (XML, annotations, or Java-based).

    The container is represented by interfaces such as BeanFactory and ApplicationContext.

    6. What is the difference between BeanFactory and ApplicationContext in Spring?

    Answer:

    • BeanFactory: The core container in Spring responsible for basic DI functionalities. It lazily initializes beans (i.e., creates beans only when requested). It is lightweight but lacks advanced features.
    • ApplicationContext: Extends BeanFactory and provides additional enterprise-level features like:
      • Eager initialization of singleton beans.
      • Event propagation mechanism.
      • Internationalization (i18n) support.
      • Convenient access to application resources (e.g., files).
      • Aware interfaces such as ApplicationContextAware.
    • ApplicationContext is more commonly used for modern Spring applications.

    7. What are the types of Dependency Injection in Spring?

    Answer:

    • Constructor Injection: Dependencies are injected through a constructor. It ensures that dependencies are provided at object creation time, making the object immutable. Example:
      • @Autowired
      public MyClass(MyService myService) {
          this.myService = myService;
      }
    • Setter Injection: Dependencies are injected through setter methods. It allows setting dependencies after object creation, which is useful for optional dependencies. Example:
      • @Autowired
      public void setMyService(MyService myService) {
          this.myService = myService;
      }

    8. What is the difference between @Component, @Service, @Repository, and @Controller?

    Answer:

    • @Component: Generic stereotype for any Spring-managed component.
    • @Service: Indicates the class contains business logic.
    • @Repository: Indicates the class is a Data Access Object (DAO) and provides additional support for exception translation.
    • @Controller: Used to define a controller in Spring MVC.

    9. What is the role of the @Autowired annotation in Spring?

    Answer: @Autowired is used to automatically inject dependencies in Spring. When Spring encounters @Autowired, it tries to resolve the dependency by:

    • Matching beans by type (default).
    • Optionally using qualifiers to inject a specific bean.
    • Autowiring constructor, fields, or setter methods.

    Example:

    @Autowired
    private MyService myService;

    If no matching bean is found or if multiple beans are found, Spring will throw an exception unless you specify required=false.

    10. What is the @Qualifier annotation, and how is it used in Spring?

    Answer: @Qualifier is used in conjunction with @Autowired to resolve ambiguity when multiple beans of the same type exist. It allows you to specify which bean should be injected.

    Example:

    @Autowired
    @Qualifier("specificService")
    private MyService myService;

    Here, @Qualifier ensures that the bean named specificService is injected into myService.

    11. Can we inject beans into another bean in Spring without @Autowired or XML?

    Answer: Yes, beans can be injected using constructor-based dependency injection through Java configuration without using @Autowired or XML configuration. This is done in a class annotated with @Configuration.

    Example:

    @Configuration
    public class AppConfig {
        @Bean
        public MyService myService() {
            return new MyServiceImpl();
        }

        @Bean
        public MyController myController(MyService myService) {
            return new MyController(myService);
        }
    }

    In this example, MyController is injected with MyService via the constructor in Java configuration.

    12. What is a prototype bean scope in Spring?

    Answer: In Spring, the prototype scope means that a new instance of the bean will be created every time it is requested from the container. This is different from the default singleton scope where only one instance is created.

    Example:

    @Bean
    @Scope("prototype")
    public MyBean myBean() {
        return new MyBean();
    }

    In this case, every call to myBean() will return a new instance of MyBean.

    13. What is the default scope of a Spring bean?

    Answer: The default scope of a Spring bean is singleton. This means that a single instance of the bean is created by the Spring IoC container, and this instance is shared across the entire application.

    Example:

    @Bean
    public MyBean myBean() {
        return new MyBean();
    }

    Even without explicitly specifying a scope, this bean will have a singleton scope by default.

    14. What is the use of the @Primary annotation in Spring?

    Answer: @Primary is used to define the default bean when multiple beans of the same type exist. If you don’t explicitly specify which bean to inject using @Qualifier, Spring will use the bean marked with @Primary.

    Example:

    @Primary
    @Bean
    public MyService primaryService() {
        return new MyServiceImpl();
    }

    In this example, primaryService() will be the default service when multiple services of the same type are available.

    15. What is the @Configuration annotation in Spring?

    Answer: @Configuration is an annotation that marks a class as a source of bean definitions. Classes annotated with @Configuration can declare one or more @Bean methods, which Spring will use to manage and configure beans.

    Example:

    @Configuration
    public class AppConfig {
        @Bean
        public MyService myService() {
            return new MyServiceImpl();
        }
    }

    This class is equivalent to an XML-based configuration file.

    16. What is the difference between XML-based configuration and annotation-based configuration in Spring?

    Answer:

    • XML-based configuration: Beans and their dependencies are defined in XML files. This was the traditional way of configuring Spring applications but has become less popular in favor of annotations and Java-based configuration. Example:
      • <bean id "myService" class="com.example.MyServiceImpl"/>
    • Annotation-based configuration: Beans are defined using annotations like @Component, @Service, @Repository, and dependencies are injected using @Autowired. This approach is concise and reduces the need for external configuration files. Example:
      • @Component
      public class MyService {
          // Bean definition via annotation
      }

    17. What is the @Bean annotation, and how does it work?

    Answer: The @Bean annotation is used to declare a bean in Spring using Java-based configuration. It tells Spring to manage the lifecycle of the object returned by the method.

    Example:

    @Configuration
    public class AppConfig {
        @Bean
        public MyService myService() {
            return new MyServiceImpl();
        }
    }

    In this example, myService will be managed by the Spring container as a Spring bean.

    18. What is Spring’s @ComponentScan, and how is it used?

    Answer: @ComponentScan is used to specify the base packages where Spring should search for classes annotated with @Component, @Service, @Repository, or @Controller. It enables automatic detection and registration of beans.

    Example:

    @Configuration
    @ComponentScan(basePackages="com.example.services")
    public class AppConfig {
        // Configuration class
    }

    This tells Spring to scan the package com.example.services and register any annotated classes as beans.

    19. What is the difference between setter injection and constructor injection?

    Answer:

    • Constructor Injection:
      • Dependencies are injected via the constructor.
      • It ensures that the object is created with all required dependencies.
      • Best for mandatory dependencies since the object cannot be created without the required arguments.
      • Encourages immutability by preventing setter methods.
      • Example:
        • @Autowired
        public MyService(MyRepository repository) {
            this.repository = repository;
        }
    • Setter Injection:
      • Dependencies are injected via setter methods.
      • Allows optional dependencies to be set after object creation.
      • Provides flexibility in modifying the object state after creation.
      • Can lead to a mutable object design.
      • Example:
        • @Autowired
        public void setRepository(MyRepository repository) {
            this.repository = repository;
        }

    20. What are the main differences between @Component, @Service, @Repository, and @Controller?

    Answer:

    • @Component: A generic stereotype annotation used to mark a class as a Spring-managed component (bean). It can be used for any Spring bean.
    • @Service: A specialization of @Component, typically used to denote service layer components that contain business logic.
    • @Repository: A specialization of @Component, used to indicate DAO (Data Access Object) or repository classes that interact with the database. It also provides exception translation.
    • @Controller: A specialization of @Component, used in the presentation layer for handling web requests (typically in Spring MVC).

    All these annotations are functionally similar and are used to automatically register a bean in the Spring context, but they are semantically different and indicate their role in the application.

    21. What is @Required annotation in Spring, and how does it work?

    Answer: The @Required annotation is used to indicate that a particular property of a bean must be set during dependency injection, either via setter injection or constructor injection. If the property is not set, Spring will throw a BeanInitializationException.

    Example:

    public class MyBean {
        private String property;

        @Required
        public void setProperty(String property) {
            this.property = property;
        }
    }

    Note: The @Required annotation is considered obsolete and is not commonly used in modern Spring applications. Instead, constructor injection or other validation mechanisms are preferred.

    22. What is the @Lazy annotation in Spring?

    Answer: The @Lazy annotation is used to delay the initialization of a bean until it is first needed (i.e., the bean is requested). By default, Spring initializes singleton beans at the startup of the application context, but with @Lazy, you can defer this until the bean is accessed for the first time.

    Example:

    @Bean
    @Lazy
    public MyBean myBean() {
        return new MyBean();
    }

    This helps in optimizing the startup time of the application by deferring the creation of beans that might not be used immediately.

    23. What is a FactoryBean in Spring?

    Answer: A FactoryBean is a special type of bean that can be used to create other objects. When you declare a FactoryBean, the object returned by the getObject() method of the FactoryBean is used as the actual bean.

    Example:

    public class MyFactoryBean implements FactoryBean<MyBean> {
        @Override
        public MyBean getObject() {
            return new MyBean();
        }

        @Override
        public Class<?> getObjectType() {
            return MyBean.class;
        }

        @Override
        public boolean isSingleton() {
            return true;
        }
    }

    In this example, MyFactoryBean is responsible for creating instances of MyBean.

    24. What is the @PostConstruct annotation used for?

    Answer: The @PostConstruct annotation is used to define an initialization method that should be called after the bean's properties have been set and before the bean is made available for use. It is used to perform any initialization tasks that need to be done after dependency injection.

    Example:

    public class MyBean {
        @PostConstruct
        public void init() {
            // Initialization code here
        }
    }

    In this example, the init() method will be called after the bean's properties have been set.

    25. What is the @PreDestroy annotation used for?

    Answer: The @PreDestroy annotation is used to define a method that should be called just before the bean is destroyed. It is used to release any resources or perform cleanup tasks before the bean is removed from the container.

    Example:

    public class MyBean {
        @PreDestroy
        public void cleanup() {
            // Cleanup code here
        }
    }

    In this example, the cleanup() method will be called before the bean is destroyed.

    26. What is the @Scope annotation in Spring?

    Answer: The @Scope annotation is used to define the scope of a bean, which determines the lifespan and visibility of the bean within the Spring container.

    Common scopes include:

    • Singleton: Only one instance of the bean is created per Spring container (default scope).
    • Prototype: A new instance is created every time the bean is requested.
    • Request: A new instance is created for each HTTP request (typically used in web applications).
    • Session: A new instance is created for each HTTP session (typically used in web applications).

    Example:

    @Scope("prototype")
    public class MyBean {
        // Bean code here
    }

    In this example, the MyBean class is defined with a prototype scope, meaning a new instance will be created each time it is requested.

    27. What is the difference between @Bean and @Component?

    Answer:

    • @Bean: Used in Java-based configuration to define a bean explicitly. It is typically used in configuration classes annotated with @Configuration to declare beans that should be managed by the Spring container.
    • @Component: Used to mark a class as a Spring-managed component (bean). It is a class-level annotation that indicates that the class is a candidate for auto-detection when using annotation-based configuration.

    Example of @Bean:

    @Bean
    public MyService myService() {
        return new MyServiceImpl();
    }

    Example of @Component:

    @Component
    public class MyService {
        // Bean code here
    }

    In summary, @Bean is used in configuration classes to define beans explicitly, while @Component is used to mark classes as candidates for auto-detection as Spring beans.

    28. What is Spring’s @Value annotation used for?

    Answer: The @Value annotation is used to inject values into fields, methods, or constructor parameters from property files or expression literals. It allows external configuration to be injected into Spring beans.

    Example:

    @Value("${app.name}")
    private String appName;
    public void setAppName(String appName) {
        this.appName = appName;
    }

    In this example, the value of app.name from the property file will be injected into the appName field.

    29. What is Spring’s @Profile annotation used for?

    Answer: The @Profile annotation is used to define beans that should be loaded or activated only for specific environments or profiles. It allows you to create beans that are only available in certain conditions, such as different environments (development, test, production).

    Example:

    @Profile("dev")
    @Bean
    public MyBean myBean() {
        return new MyBean();
    }

    In this example, the MyBean bean will only be created if the "dev" profile is active.

    30. What is the difference between @Component and @Service?

    Answer: Both @Component and @Service are used to mark a class as a Spring-managed component, but they are used for different purposes:

    • @Component: A generic stereotype annotation for any Spring-managed component. It is used when the class does not fit into other more specific roles.
    • @Service: A specialization of @Component used to denote service layer components that contain business logic. It is a more specific role compared to @Component and can be used to indicate the business logic layer.

    Both annotations function similarly in terms of bean registration, but @Service provides a more specific semantic meaning in the application context.
