    Java Hash Interview Questions

    1. How does a HashMap work in Java?

    HashMap in Java: HashMap is a data structure that stores key-value pairs. It uses hashing to store elements and provides average O(1) time complexity for put and get operations.

    When you put a key-value pair into a HashMap:

    • HashMap calculates the hash code of the key.
    • Using this hash code, it determines the index to store the key-value pair in an array (bucket).
    • If multiple keys hash to the same index (collision), HashMap uses a linked list (or tree in Java 8+) to store these key-value pairs.
    • To retrieve a value based on a key, HashMap calculates the hash code again, finds the appropriate bucket, and then compares keys using equals() method to find the exact key-value pair.


    2. What is the distinction between HashMap and HashTable in the Java programming language?

    HashMap vs HashTable:

    • Thread safety: HashMap is not synchronized and is not thread-safe. HashTable is synchronized and is thread-safe.
    • Null keys and values: HashMap allows one null key and any number of null values. HashTable does not allow null keys or values.
    • Performance: HashMap is generally faster than HashTable for single-threaded operations due to lack of synchronization.
    • Iterators: Iterator in HashMap is fail-fast while the enumerator for HashTable is not.

    4. How does the load factor of a HashMap affect its performance?

    Load factor: Determines when to resize the HashMap. It represents how full the HashMap can be before it needs to increase its capacity.

    A smaller load factor means less collision but higher space overhead. A larger load factor means more collision but lower space overhead.

    Typical load factor is 0.75. Higher load factors improve space efficiency but increase the chances of collisions.


    5. How does the initial capacity of a HashMap affect its performance?

    Initial capacity: Specifies the initial size of the HashMap. A larger initial capacity reduces the number of rehash operations but increases memory consumption initially.

    Choosing an appropriate initial capacity can improve performance by reducing the frequency of resizing.

    1. How does a HashMap handle collision in Java?

    HashMap handles collisions by using a technique called chaining. In case of a collision (i.e., when multiple keys hash to the same index), HashMap stores these keys and their corresponding values as a linked list at that index in the underlying array. When retrieving a value for a key, HashMap uses the key's hash code to find the correct bucket (index) and then compares the keys using their equals() method to find the exact match within the linked list.

    20. Implement a function longestSubstring(String s) that takes a string as input and returns the length of the longest substring without repeating characters. 
    
							public int longestSubstring(String s) {
							    if (s == null || s.isEmpty()) return 0;
							    Map<Character, Integer> charMap = new HashMap<>();
							    int maxLength = 0;
							    for (int start = 0, end = 0; end < s.length(); end++) {
							        char currentChar = s.charAt(end);
							        if (charMap.containsKey(currentChar)) {
							            start = Math.max(charMap.get(currentChar) + 1, start);
							        }
							        charMap.put(currentChar, end);
							        maxLength = Math.max(maxLength, end - start + 1);
							    }
							    return maxLength;
							}

							public int countPairs(int[] nums, int target) {
							    Map<Integer, Integer> numCountMap = new HashMap<>();
							    int pairCount = 0;
							    for (int num : nums) {
							        int complement = target - num;
							        if (numCountMap.containsKey(complement)) {
							            pairCount += numCountMap.get(complement);
							        }
							        numCountMap.put(num, numCountMap.getOrDefault(num, 0) + 1);
							    }
							    return pairCount;
							}

    10. Implement a function palindromePairs(List<String> words) that takes a list of strings as input and returns a list of pairs of strings from the input list that can be concatenated to form a palindrome. 
    
							public List<List<String>> palindromePairs(List<String> words) {
							    List<List<String>> result = new ArrayList<>();
							    Map<String, Integer> wordIndexMap = new HashMap<>();
							    for (int i = 0; i < words.size(); i++) {
							        wordIndexMap.put(words.get(i), i);
							    }
							    for (int i = 0; i < words.size(); i++) {
							        String word = words.get(i);
							        for (int j = 0; j <= word.length(); j++) {
							            String left = word.substring(0, j);
							            String right = word.substring(j);
							            if (isPalindrome(left)) {
							                String reversedRight = new StringBuilder(right).reverse().toString();
							                if (wordIndexMap.containsKey(reversedRight) && wordIndexMap.get(reversedRight) != i) {
							                    result.add(Arrays.asList(reversedRight, word));
							                }
							            }
							            if (!right.isEmpty() && isPalindrome(right)) {
							                String reversedLeft = new StringBuilder(left).reverse().toString();
							                if (wordIndexMap.containsKey(reversedLeft) && wordIndexMap.get(reversedLeft) != i) {
							                    result.add(Arrays.asList(word, reversedLeft));
							                }
							            }
							        }
							    }
							    return result;
							}
							
							private boolean isPalindrome(String s) {
							    int left = 0;
							    int right = s.length() - 1;
							    while (left < right) {
							        if (s.charAt(left++) != s.charAt(right--)) {
							            return false;
							        }
							    }
							    return true;
							}

    Designing a scalable and maintainable HashMap implementation in Java involves considerations such as concurrency, performance, and extensibility:

    • Choose appropriate synchronization mechanisms like ConcurrentHashMap for concurrency.
    • Optimize initial capacity and load factor to balance memory usage and performance.
    • Implement eviction policies for managing memory usage, e.g., LRU (Least Recently Used) caching.
    • Support customization through interfaces for serializers, transformers, and filters.
    • Use design patterns like factory method for object creation and decorator for additional behaviors.
    • Document and test thoroughly to ensure reliability, scalability, and maintainability.
