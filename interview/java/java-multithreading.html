    Java MultiThreading Interview Questions

    1. What do you mean by a “thread” in Java programming?

    A thread in Java refers to a lightweight subprocess that is a part of a larger process. Threads allow concurrent execution of tasks within a single program, enabling better utilization of CPU resources and improved responsiveness.


    2. What are some fundamental advantages of multithreading in Java?

    Some advantages of multithreading in Java include:

    • Improved responsiveness: Allows programs to remain interactive during long-running tasks.
    • Resource sharing: Threads can share resources like memory, reducing overhead.
    • Efficient utilization of CPU: Allows concurrent execution of tasks, maximizing CPU usage.
    • Modular and simpler code: Threads enable modular design for complex tasks.

    3. What is the basic difference between a process and a thread?

    A process is an independent program that runs in its own memory space and is managed by the operating system. A thread, on the other hand, is a lightweight unit of a process and shares the process's memory and resources.


    4. What is inter-thread communication for threads?

    Inter-thread communication refers to mechanisms used by threads to synchronize their actions or exchange data. It allows threads to coordinate and cooperate towards completing a task efficiently.


    5. What are some functions that are used for inter-thread communication in Java?

    In Java, inter-thread communication can be achieved using methods such as wait(), notify(), and notifyAll() from the Object class. These methods are used to manage thread synchronization and data sharing.


    6. What do you understand by the wait() function?

    The wait() function in Java is used for inter-thread communication. It causes the current thread to wait until another thread invokes notify() or notifyAll() methods for the same object, or a specified amount of time has elapsed.


    7. What are the states in a thread lifecycle?

    The states in a thread lifecycle in Java are:

    • New: When a thread is created but not yet started.
    • Runnable: When a thread is ready to run and waiting for CPU time.
    • Blocked: When a thread is waiting for a monitor lock to enter a synchronized block/method.
    • Waiting: When a thread is waiting indefinitely for another thread's signal (via wait()).
    • Timed Waiting: When a thread is waiting for a specified time period (via sleep() or wait(timeout)).
    • Terminated: When a thread completes its execution or is terminated.

    8. What do you understand by thread context switching?

    Thread context switching refers to the process where the CPU switches from executing one thread to another. It involves saving and restoring the state of a thread so that it can be resumed later.


    9. What is the function of the join() method?

    The join() method in Java allows one thread to wait until another thread completes its execution. It causes the current thread to pause execution until the thread on which join() was called terminates.


    10. What is the function of the sleep() method?

    The sleep() method in Java pauses the execution of the current thread for a specified amount of time, allowing other threads to execute. It is useful for introducing delays or for time-based operations.


    11. What is a Process and process control block?

    A process in computing is an instance of a program being executed. A process control block (PCB) is a data structure used by the operating system to manage each process. It contains process state, program counter, stack pointer, and other relevant information.


    12. What is a Thread?

    A thread is a lightweight subprocess within a process that can be scheduled independently by the operating system's scheduler. Threads share the same memory space and resources within a process.


    13. What is a Practical Example of a Process and a Thread?

    An example of a process is a web browser running on your computer. Within the web browser, each tab may be managed by a separate thread. Thus, the web browser process consists of multiple threads handling different tasks like rendering, networking, and user input.


    14. What are the Differences Between Process and Thread?

    Some key differences between a process and a thread include:

    • Memory: Processes have separate memory spaces, while threads share memory within the same process.
    • Overhead: Creating a process is more resource-intensive than creating a thread.
    • Independence: Processes are independent and isolated, while threads share resources like memory and files.
    • Communication: Inter-process communication is more complex than inter-thread communication.

    15. Explain the Differences Between Process and Thread Context Switching?

    Context switching for processes involves switching the entire process state, including memory maps and resource allocation. For threads, context switching involves switching the execution state and program counter within the same process memory space.


    16. List the difference between preemptive scheduling and time slicing?

    In preemptive scheduling, the operating system decides when to switch threads based on priority and time quantum, interrupting threads as necessary. Time slicing is a form of preemptive scheduling where each thread is allocated a fixed time slice to execute before switching to another thread.


    17. What is context switching in multi-process or multithreading environment?

    Context switching refers to the process of saving and restoring the state of a CPU so that multiple processes or threads can share the same CPU. It involves storing the current state of a thread or process and loading the saved state of another.


    18. How to Create a Thread Instance and Run It?

    You can create a thread instance in Java by extending the Thread class or implementing the Runnable interface. Here's an example using Runnable:

							    public void run() {
							        System.out.println("Thread is running.");
							    }
							
							    public static void main(String[] args) {
							        MyRunnable myRunnable = new MyRunnable();
							        Thread thread = new Thread(myRunnable);
							        thread.start(); // Start the thread
							    }
							}

    19. List the Different States of a Thread and When Do the State Transitions Occur.

    The different states of a thread in Java are:

    • New: When a thread instance is created but not yet started.
    • Runnable: When a thread is ready to run and waiting for CPU time.
    • Blocked: When a thread is waiting for a monitor lock to enter a synchronized block/method.
    • Waiting: When a thread is waiting indefinitely for another thread's signal (via wait()).
    • Timed Waiting: When a thread is waiting for a specified time period (via sleep() or wait(timeout)).
    • Terminated: When a thread completes its execution or is terminated.

    State transitions occur based on thread scheduling and synchronization operations such as acquiring locks, waiting for signals, and completing execution.


    20. Explain the Difference Between Runnable and Callable Interfaces? How Are They Used?

    The Runnable interface is used to define a task that can be executed by a thread. It does not return a result or throw a checked exception. The Callable interface is similar but can return a result and throw a checked exception.

    Here's an example:

    							            import java.util.concurrent.Callable;
							import java.util.concurrent.ExecutorService;
							import java.util.concurrent.Executors;
							import java.util.concurrent.Future;
							
							public class MyCallable implements Callable<String> {
							    public String call() throws Exception {
							        return "Callable task is running.";
							    }
							
							    public static void main(String[] args) throws Exception {
							        ExecutorService executor = Executors.newSingleThreadExecutor();
							        MyCallable myCallable = new MyCallable();
							        Future<String> future = executor.submit(myCallable);
							        System.out.println(future.get()); // Print result of callable task
							        executor.shutdown(); // Shutdown executor
							    }
							}

    21. Explain the Daemon Thread, and What Are Its Use Cases? How Can You Create a Daemon Thread?

    A daemon thread in Java is a background thread that runs without preventing the JVM from exiting. Daemon threads are used for tasks like garbage collection, monitoring, or other background tasks that do not require user interaction or continuation after main threads have completed.

    You can create a daemon thread by calling the setDaemon(true) method before starting the thread:

    							            public class MyDaemonThread extends Thread {
							    public void run() {
							        while (true) {
							            System.out.println("Daemon thread is running.");
							        }
							    }
							
							    public static void main(String[] args) {
							        MyDaemonThread daemonThread = new MyDaemonThread();
							        daemonThread.setDaemon(true); // Set daemon thread
							        daemonThread.start(); // Start daemon thread
							    }
							}

    22. What are differences between the wait() and sleep() methods?

    The wait() method is used for inter-thread communication and is called on an object's monitor within synchronized blocks. It causes the current thread to wait until another thread notifies or interrupts it.

    The sleep() method is used to pause the execution of the current thread for a specified amount of time, allowing other threads to execute. It does not release the lock on the object.


    23. List down the difference between class level lock and object level lock?

    In Java, synchronization using the synchronized keyword can be applied at the class level or the object level:

    • Object level lock: Synchronizing on an instance method or block uses the current instance (this) as the lock.
    • Class level lock: Synchronizing on a static method or block uses the class itself as the lock.

    Object level locks synchronize access to instance variables and methods for a specific instance, while class level locks synchronize access to static variables and methods for the entire class.


    24. What's the difference between a User thread and a Daemon thread?

    A user thread in Java is a thread created by the application developer and does not prevent the JVM from exiting until all user threads have completed. A daemon thread, on the other hand, runs in the background and does not prevent the JVM from exiting if all user threads have completed.


    25. How can we create daemon threads?

    You can create a daemon thread in Java by calling the setDaemon(true) method on a thread object before starting it. Daemon threads should be created before starting the thread:

							        // Daemon thread logic
							    }
							
							    public static void main(String[] args) {
							        MyDaemonThread daemonThread = new MyDaemonThread();

    26. Explain volatile variables in Java?

    In Java, the volatile keyword is used to indicate that a variable's value may be modified by multiple threads. It ensures that reads and writes to the variable are atomic and visible to other threads immediately, without caching.


    27. How do threads communicate with each other?

    Threads communicate with each other through shared memory or message passing mechanisms like wait/notify, synchronized blocks, locks, semaphores, and queues. These mechanisms enable threads to coordinate their activities and share data safely.


    28. Can two threads execute two methods (static and non-static concurrently)?

    Yes, two threads can execute two methods concurrently, whether they are static or non-static methods. The concurrency depends on thread scheduling and synchronization mechanisms used to control access to shared resources.


    29. What is the purpose of the finalize() method?

    The finalize() method in Java is called by the garbage collector before reclaiming the memory occupied by an object that is no longer referenced. It can be overridden to perform cleanup operations or release resources before an object is garbage collected.


    30. Is it possible to call the run() method directly to start a new thread?

    Yes, it is possible to call the run() method directly, but it will execute in the current thread, not as a separate thread. To start a new thread and execute its run() method concurrently, you should call the start() method.


    31. Is it possible that each thread can have its stack in multithreaded programming?

    Yes, each thread in Java has its own stack memory allocated. The stack holds method calls, local variables, and partial results, ensuring thread safety and isolation.


    32. What do you understand by inter-thread communication?

    Inter-thread communication refers to mechanisms used by threads to synchronize their actions or exchange data. It enables threads to coordinate their activities, share resources, and communicate effectively to achieve synchronization and mutual exclusion.


    33. Why does the wait() method need to be called from a synchronized block?

    The wait() method in Java must be called from within a synchronized block or method because it releases the lock on the object it is called on, allowing other threads to acquire the lock and invoke notify() or notifyAll() to wake up the waiting thread.

    35. Name the method of the thread that is called before the run() method and carries out initialization.

    The start() method of the thread is called before the run() method. The start() method initializes the thread, schedules it for execution, and then calls the run() method.


    36. How do you define Multithreading?

    Multithreading is a programming concept where multiple threads within a single process execute independently and concurrently. Threads share the same memory space and resources of their parent process, enabling efficient resource utilization and concurrent task execution.


    37. Name the method that is used to register a thread in a thread scheduler.

    The start() method of the Thread class is used to register a thread with the thread scheduler. It allocates necessary resources and schedules the thread for execution by invoking its run() method.


    38. What is the type of thread priority in Java?

    In Java, threads can have priorities ranging from 1 (lowest) to 10 (highest). Thread priority helps the thread scheduler determine the order of thread execution when multiple threads are ready to run.


    1. Explain meaning of the Thread’s Interrupt Flag? How Can You Set and Check It? How Does It Relate to the Interrupted exception?

    The interrupt flag in a thread is a boolean attribute that indicates whether the thread has been interrupted or not. You can set this flag using the interrupt() method of the thread instance. To check if a thread's interrupt flag has been set, you can use the isInterrupted() method. If a thread is interrupted while it's in a blocking operation (like wait(), sleep(), or I/O operations), it can throw an InterruptedException.


    2. What is an executor and an executor service? What are the differences between these interfaces?

    An executor in Java is an object that manages the execution of threads. It decouples task submission from task execution, allowing you to control the execution of asynchronous tasks. An executor service extends this concept by providing a higher-level interface for managing thread execution and lifecycle, such as task scheduling, execution, and termination.

    The main difference is that an executor simply executes submitted tasks, while an executor service provides methods to manage and control tasks, including shutdown and termination.


    3. What are the ExecutorService implementations in the standard library?

    In the standard Java library, some implementations of ExecutorService include:

    • ThreadPoolExecutor
    • ScheduledThreadPoolExecutor
    • ForkJoinPool

    4. What is Java Memory Model (JMM)? Describe its Purpose and Basic Ideas.

    The Java Memory Model (JMM) defines how threads interact through memory. It ensures that changes made by one thread to shared variables are visible to other threads. JMM guarantees that certain operations are atomic and defines the order of operations in a multi-threaded environment.


    5. What is a Volatile Field and What Guarantees Does the JMM Hold for Such Field?

    A volatile field in Java is a variable whose value is always read from and written to main memory (not cached thread-locally). The JMM guarantees that reads and writes to a volatile variable are atomic and that changes to the volatile variable are visible to all threads immediately after the modification.


    6. Which of the Following Operations Are Atomic?

    Atomic operations in Java are operations that execute completely and without interruption, such as:

    • Reads and writes to primitive variables (e.g., int, long)
    • Reads and writes to volatile variables
    • Operations on AtomicInteger, AtomicLong, etc.

    7. What Is the Meaning of a Synchronized Keyword in the Definition of a Method? of a Static Method? Before a Block?

    The synchronized keyword in Java is used to achieve synchronization, ensuring that only one thread can execute a synchronized block of code or method at a time:

    • Method: Synchronizes the entire method, preventing multiple threads from executing it concurrently for the same instance.
    • Static Method: Synchronizes the entire static method, preventing multiple threads from executing it concurrently for the same class.
    • Block: Synchronizes a specific block of code, allowing fine-grained control over synchronization on a particular object's monitor.

    8. If Two Threads Call a Synchronized Method on Different Object Instances Simultaneously, Could One of These Threads Block? What If the Method Is Static?

    If two threads call a synchronized method on different object instances simultaneously, they won't block each other because they are locking on different objects. However, if the synchronized method is static, it locks on the class object (class-level lock), so even different instances will block each other.


    9. What is the Purpose of the Wait, Notify, and Notifyall Methods of the Object Class?

    These methods are used for inter-thread communication in Java:

    • wait(): Causes the current thread to wait until another thread invokes notify() or notifyAll() on the same object.
    • notify(): Wakes up a single thread that is waiting on this object.
    • notifyAll(): Wakes up all threads that are waiting on this object.

    10. Describe the Conditions of Deadlock, Livelock, and Starvation. Describe the Possible Causes of These Conditions.

    Deadlock: Occurs when two or more threads are blocked forever, each waiting on a resource held by the other.

    Livelock: Occurs when two or more threads keep responding to each other's actions without making progress.

    Starvation: Occurs when a thread is unable to gain regular access to resources and is unable to make progress.

    These conditions are usually caused by improper synchronization, incorrect locking order, or inefficient resource management.


    11. Describe the Purpose and Use-Cases of the Fork/Join Framework.

    The Fork/Join framework in Java provides support for parallel programming by efficiently utilizing multiple processors. It is particularly useful for recursive tasks that can be divided into smaller tasks and executed concurrently.

    Use cases include parallel processing of large datasets, recursive algorithms like merge sort or Fibonacci sequence calculation, and tasks that benefit from multi-threaded execution.


    12. Why wait(), notify(), and notifyAll() methods are present in Object class?

    These methods are present in the Object class to provide a mechanism for threads to communicate and synchronize their actions on shared resources. They enable efficient inter-thread communication and coordination without the need for explicit thread management by the application developer.


    13. What is ReentrantLock in Java Multi-Threading?

    ReentrantLock is a synchronization mechanism in Java that implements the Lock interface. It provides more flexibility and control over the locking mechanism compared to synchronized blocks or methods. It supports features like fairness policies, interruptible lock acquisition, and non-blocking tries to acquire locks.


    14. What is Thread Stack and Heap Memory?

    In Java, each thread has its own thread stack and shares the heap memory with other threads. The thread stack contains method calls, local variables, and partial results specific to that thread. Heap memory contains objects and is shared across all threads in the JVM.


    15. Why are Local Variables Thread-safe in Java?

    Local variables in Java are thread-safe because they are stored on the stack memory, which is private to each thread. Each thread has its own stack memory, so local variables are not shared and cannot be accessed or modified by other threads.


    16. What is the start() and run() method of Thread class?

    The start() method is used to begin execution of the thread and calls the run() method asynchronously. The run() method contains the code that executes in the thread and should be overridden to define the task that the thread performs.


    17. Explain thread pool?

    A thread pool in Java is a collection of pre-initialized threads that are ready to execute tasks. It manages the execution of tasks asynchronously, reusing threads from the pool to improve performance and efficiency by reducing thread creation overhead.


    18. What’s the purpose of the join() method?

    The join() method in Java allows one thread to wait for the completion of another thread. When a thread calls join() on another thread, it suspends its own execution until the target thread has completed its execution.


    19. What do you mean by garbage collection?

    Garbage collection in Java is the process of automatically reclaiming memory occupied by objects that are no longer reachable or in use by the application. It helps manage memory efficiently and eliminates memory leaks by freeing up resources for reuse.


    20. What is the lock interface? Why is it better to use a lock interface rather than a synchronized block.?

    The Lock interface in Java provides a more flexible and powerful mechanism for synchronization compared to synchronized blocks:

    • It supports features like timed lock acquisition, interruptible lock acquisition, and multiple condition variables.
    • It allows better control over lock release and avoids potential issues like deadlock.
    • It offers better performance in scenarios with frequent contention and fine-grained locking requirements.

    21. What do you understand about Java Multithreading deadlocks?

    Deadlock in Java occurs when two or more threads are blocked forever, each waiting for a resource held by the other thread(s). It typically happens when threads acquire multiple locks in different order or when circular dependencies occur in resource allocation.


    22. How to identify a deadlock in Java?

    Deadlocks in Java can be identified using thread dump analysis tools like jstack or by examining application logs and monitoring tools that indicate threads waiting for locks that are held by other threads indefinitely.


    23. How can deadlock be avoided in Java?

    Deadlock prevention in Java can be achieved by:

    • Ensuring a consistent and predefined order for acquiring locks.
    • Using timeout mechanisms with locks to avoid indefinite waiting.
    • Limiting the scope and duration of lock acquisition.
    • Using java.util.concurrent locks and utilities that offer advanced features for deadlock avoidance.

    24. Does a thread have its own stack in Multithreaded programming?

    Yes, each thread in Java has its own thread stack memory. The stack memory holds method calls, local variables, and partial results specific to that thread. This isolation ensures thread safety and prevents interference between threads.


    25. How is thread-safety achieved in multithreaded programming?

    Thread-safety in Java is achieved through synchronization mechanisms like:

    • synchronized blocks and methods
    • volatile variables
    • java.util.concurrent locks and classes (e.g., ReentrantLock, Atomic classes)
    • Immutable objects
    • Thread-safe data structures (e.g., ConcurrentHashMap, BlockingQueue)

    26. What is an atomic operation? What are atomic classes in Java Concurrency API?

    An atomic operation in Java is an operation that executes as a single, indivisible unit without interference from other threads. Atomic classes in Java Concurrency API, like AtomicInteger, AtomicLong, and AtomicReference, provide atomic operations on these types, ensuring thread-safe access without using synchronization.


    27. What is the Lock interface in Java Concurrency API? What are its benefits over synchronization?

    The Lock interface in Java Concurrency API provides a more flexible and powerful alternative to synchronized blocks:

    • It supports features like timed lock acquisition, interruptible lock acquisition, and multiple condition variables.
    • It allows better control over lock release and avoids potential issues like deadlock.
    • It offers better performance in scenarios with frequent contention and fine-grained locking requirements.

    28. What is Executors Framework?

    The Executors framework in Java provides a higher-level API for managing and controlling thread execution. It simplifies the task of creating and managing thread pools, asynchronous tasks, and concurrent computations.


    29. What is Callable and Future?

    Callable is a functional interface in Java that represents a task that can be executed asynchronously. It is similar to Runnable but can return a result and throw a checked exception.

    Future is an interface that represents the result of an asynchronous computation. It provides methods to check if the computation is complete, retrieve the result, and cancel the task.


    30. Explain context switching.

    Context switching is the process of saving and restoring the state of a thread or process so that it can be scheduled and executed later. It allows the CPU to switch from executing one thread to another, enabling multitasking and concurrent execution of multiple threads.


    31. What is FutureTask Class?

    FutureTask is a concrete implementation of the Future interface that represents an asynchronous computation with a result. It encapsulates the computation and provides methods to start, cancel, and retrieve the result of the computation.


    32. What is BlockingQueue? How it is used with producer and consumer thread?

    BlockingQueue is a type of queue in Java that supports operations that wait for the queue to become non-empty when retrieving an element and wait for space to become available in the queue when storing an element. It is commonly used to implement producer-consumer scenarios, where multiple threads produce data (producers) and consume data (consumers) concurrently.


    33. What is CyclicBarrier and CountDownLatch?

    CyclicBarrier and CountDownLatch are synchronization aids in Java:

    • CyclicBarrier allows a set of threads to wait for each other to reach a common barrier point before continuing execution.
    • CountDownLatch allows one or more threads to wait until a set of operations being performed in other threads completes.

    34. What is Executors Class?

    The Executors class in Java provides factory and utility methods for creating and managing thread pools, asynchronous tasks, and executing tasks with various scheduling policies. It simplifies the task of working with threads and managing concurrency in Java applications.


    35. What are some of the improvements in Concurrency API in Java 8?

    Java 8 introduced several enhancements to the Concurrency API, including:

    • New methods in CompletableFuture for combining and handling asynchronous tasks.
    • Enhancements to ConcurrentHashMap with new methods for parallel processing and bulk operations.
    • Addition of StampedLock for optimistic locking.
    • Improved support for parallel streams and operations on collections.

    36. What is IPC and how do the threads communicate with each other?

    IPC (Inter-Process Communication) is a mechanism that allows processes or threads to communicate and synchronize their actions in a shared memory space. In Java, threads communicate with each other using synchronization primitives like locks, semaphores, monitors, and higher-level constructs like wait(), notify(), and notifyAll() methods.


    37. What is ConcurrentHashMap and Hashtable? In java, why is ConcurrentHashMap considered faster than Hashtable?

    ConcurrentHashMap and Hashtable are both thread-safe implementations of the Map interface in Java:

    • Hashtable is synchronized, which means all methods are thread-safe, but it can lead to contention under heavy concurrent access.
    • ConcurrentHashMap provides better concurrency by allowing concurrent reads and updates without locking the entire map, using finer-grained locking and partitioning.
    • Due to its design, ConcurrentHashMap is considered faster than Hashtable in scenarios with high concurrency.

    38. What is semaphore?

    A semaphore in Java is a synchronization aid that allows a fixed number of threads to access a shared resource concurrently. It maintains a set of permits, where each acquire() operation blocks if necessary until a permit is available, and each release() operation adds a permit, potentially releasing a blocking acquirer.


    39. What will happen if we don’t override the thread class run() method?

    If you don't override the run() method in a subclass of Thread, the default implementation from the Thread class will be executed. This default implementation does nothing, so the thread will terminate immediately after being started without performing any useful work.


    40. Is Multithreading present in other programming languages?

    Yes, multithreading is present in many other programming languages besides Java, including C++, Python, C#, and JavaScript. Each language provides its own mechanisms and libraries for managing threads and concurrent execution.


    8. Java Multithreading Interview Questions


    1) What is multitasking?

    A multitasking operating system is an operating system that gives you the perception of 2 or more tasks/jobs/processes running at the same time. It does this by dividing system resources amongst these tasks/jobs/processes and switching between the tasks/jobs/processes while they are executing over and over again. Usually, the CPU processes only one task at a time but the switching is so fast that it looks like the CPU is executing multiple processes at a time. They can support either preemptive multitasking, where the OS provides time to applications (virtually all modern OS), or cooperative multitasking, where the OS waits for the program to give back control (Windows 3.x, Mac OS 9, and earlier), leading to hangs and crashes. Also known as Timesharing, multitasking is a logical extension of multiprogramming.

    Multitasking programming is of two types which are as follows:

    1. Process-based Multitasking
    2. Thread-based Multitasking


    2. What is the difference between a thread and a process?

    By asking this, the interviewer wants to see that you understand fundamental multithreading concepts. Explain the concept clearly as if you are speaking to someone who does not have experience in this field.
    Example: 'A process is a single application or program, whereas a thread is a subprocess within that application or program. Each process has its own address space in memory. Threads share their address space.'

    The key benefits of Java multithreading include:

    • Multiple threads can share single address spaces
    • Threads share the same resources from the library but can execute differently
    • Different threads can be made to perform various functions to yield a given result

    4. What is inter-thread communication?

    Communication between synchronized threads is referred to as inter-thread communication. It is a core feature that helps to avoid thread-polling in Java. A particular thread can be paused through inter-thread communication to allow another thread to enter the block.


    5. What are some functions used to perform inter-thread communication in Java?

    This is one of the most common Java Multithreading interview questions asked in tech interviews. Some common functions used to perform inter-thread communication in Java are - notify(), wait() and notifyAll().


    6. What do you understand about the wait() function in Java?

    The wait() function, specified by the object class, is used to pause the current thread and wait until another thread calls the notify() function. Note that the wait() method needs to be called from a synchronized block to avoid an exception from occurring.


    7. What are the different states in a thread lifecycle?

    There are six thread states. They are NEW, RUNNABLE, BLOCKED, WAITING, TIMED_WAITING and TERMINATED. At any point of time, thread will be in any one of these states.

    1. NEW : A thread will be in this state before calling start() method.
    2. RUNNABLE : A thread will be in this state after calling the start() method.
    3. BLOCKED : A thread will be in this state when a thread is waiting for object lock to enter into synchronized method/block or a thread will be in this state if deadlock occurs.
    4. WAITING : A thread will be in this state when wait() or join() method is called.
    5. TIMED_WAITING : A thread will be in this state when sleep() or wait() with timeOut or join() with timeOut is called.
    6. TERMINATED : A thread will be in this state once it finishes it’s execution.

    8. What do you understand by context switching?

    Context switching is a feature through which the current state of a thread is saved for it to be restored and executed later. Through context switching, multiple processes can share the same CPU.


    9. What is the function of the join() method?

    The join() method causes the current thread to stop running until the thread due to congregate with completes its task. It is a commonly used function that facilitates the execution of multiple threads in an organized manner.


    10. What is the function of the sleep() method?

    11. What do you understand about Deadlock situations in Java Multithreading?

    Deadlock is a situation where each thread is waiting for resources held by other waiting threads. Due to this situation, no threads are executed, causing a pause in program execution and breaking the code at runtime.

    13. How can deadlock situations be avoided in Java?

    This is one of the most common Java Multithreading interview questions asked in technical interviews. Deadlock situations in Java can be avoided by: By way of avoiding nested thread locks and providing locks to only one thread at a time By using thread join - the function helps to wait for threads to execute before other threads are executed, thereby preventing multiple threads from waiting for resources used by other threads.

    17. How many types of threads are there in Java? Explain?

    There are two types of threads in Java. They are,

    1. User Threads
    2. Daemon Threads

    User threads are threads which are created by the application or user. They are high priority threads. JVM will not exit until all user threads finish their execution. JVM wait for user threads to finish their task. These threads are foreground threads.

    Daemon threads are threads which are mostly created by the JVM. These threads always run in background. These threads are used to perform some background tasks like garbage collection. These threads are less priority threads. JVM will not wait for these threads to finish their execution. JVM will exit as soon as all user threads finish their execution.

    19. What are MIN_PRIORITY, NORM_PRIORITY and MAX_PRIORITY?

    MIN_PRIORITY, NORM_PRIORITY and MAX_PRIORITY are three constant fields in java.lang.Thread class which define lowest, normal and highest priority of a thread respectively.

    MIN_PRIORITY : It defines the lowest priority that a thread can have and it’s value is 1.

    NORM_PRIORITY : It defines the normal priority that a thread can have and it’s value is 5.

    MAX_PRIORITY : It defines the highest priority that a thread can have and it’s value is 10.


    20. What do you mean by synchronization? Explain with an example?

    Through synchronization, we can make the threads to execute particular method or block in sync not simultaneously. When a method or block is declared as synchronized, only one thread can enter into that method or block. When one thread is executing synchronized method or block, the other threads which wants to execute that method or block have to wait until first thread executes that method or block. Thus avoiding the thread interference and achieving the thread safeness.


    21. What is object lock or monitor?

    The synchronization in Java is built around an entity called object lock or monitor. Below is the brief description about lock or monitor.

    • Whenever an object is created to any class, an object lock is created and is stored inside the object.
    • One object will have only one object lock associated with it.
    • Any thread wants to enter into synchronized methods or blocks of any object, they must acquire object lock associated with that object and release the lock after they are done with the execution.
    • The other threads which wants to enter into synchronized methods of that object have to wait until the currently executing thread releases the object lock.
    • To enter into static synchronized methods or blocks, threads have to acquire class lock associated with that class as static members are stored inside the class memory.

    23. What is the use of synchronized blocks?

    Synchronization slows down the application. Because, at any given time, only one thread can enter into synchronized method. Other threads have to wait until first thread finishes it’s execution of that method. This slows down the execution of whole application.

    Instead of synchronizing the whole method, synchronizing the only that part which is to be monitored for thread safe saves the time. This can be done using synchronized blocks.


    24. How threads communicate with each other in Java?

    Threads in Java communicate with each other using wait(), notify() and notifyAll() methods.

    wait() : This method tells the currently executing thread to release the lock of this object and wait until some other thread acquires the same lock and notify it using either notify() or notifyAll() methods.

    notify() : This method wakes up one thread randomly that called wait() method on this object.

    notifyAll() : This method wakes up all the threads that called wait() method on this object. But, only one thread will acquire lock of this object depending upon the priority.

    26. What is the difference between notify() and notifyAll() in Java?

    notify() : When a thread calls notify() method on a particular object, only one thread will be notified which is waiting for the lock or monitor of that object. The thread chosen to notify is random i.e randomly one thread will be selected for notification. Notified thread doesn’t get the lock of the object immediately. It gets once the calling thread releases the lock of that object.

    notifyAll() : When a thread calls notifyAll() method on a particular object, all threads which are waiting for the lock of that object are notified. All notified threads will move from WAITING state to BLOCKED state. All these threads will get the lock of the object on a priority basis. The thread which gets the lock of the object moves to RUNNING state. The remaining threads will remain in BLOCKED state until they get the object lock.


    27. What do you know about interrupt() method? Why it is used?

    interrupt() method is used to interrupt sleeping or waiting thread. The whole thread interruption mechanism depends on an internal flag called interrupt status. The initial value of this flag for any thread is false. When you call interrupt() method on a thread, interrupt status of that thread will be set to true. When a thread throws InterruptedException, this status will be set to false again.


    28. What is the difference between calling start() method and calling run() method directly as anyhow start() method internally calls run() method?

    When you call start() method, a new thread is created and that newly created thread executes the task kept in the run() method. If you call run() method directly, no new thread is created. Any task kept in run() method is executed by the calling thread itself.

    If you are calling run() method directly, then you are not making use of the multi-threaded programming concept. Because, when you call run() method directly, no new thread is created. run() method is executed by the calling thread itself. It just acts as normal method invocation. You are not using the concept of multi-threading.


    29. What is executor framework?

    Java 5 has introduced new framework named executor framework for managing threads.

    If you need to create thousands of the threads, your application performance may suffer and also, maintenance of each thread is also overhead. Executor framework solves this problem by limiting number of threads and reusing the same threads once threads had completed its current task.


    30. What is Thread pool?

    Thread pool represents set of worker threads that are waiting for the jobs and these worker threads can be reused many times.

    Whenever a task needs to be performance, service provider will pull out a thread out of thread pool to perform the task. If no worker threads are available, then task has to wait for execution.

    java.util.concurrent.Executors class provides factory methods to create the thread pools.


    31. Can you list down important factory methods of Executors class?

    Here are the few important methods of Executors class.

    newFixedThreadPool: This method returns a thread pool whose maximum size is fixed. If all threads are busy in execution of tasks and additional tasks are submitted, then they have to wait in queue until any thread is available for executing these tasks.

    newCachedThreadPool: This method returns unbounded thread pool. If threads are not used for certain defined time (keepAliveTime), then it will kill extra threads.

    newSingleThreadedExecutor: This method returns an Executor with single thread.

    newScheduledThreadPool: This method returns thread pool with fixed size, that can schedule task periodically or with given delay.


    32. What is BlockingQueue

    BlockingQueue is special type of queue which is used when producer threads produce object and consumer threads consume object.

    Producer thread keeps inserting objects in the queue unless it is full and once it is full, it will be blocked unless consumer threads start consuming.

    Similarly, consumer thread keep consuming objects until it is empty. Once it is empty, it will be blocked unless producer threads start producing.

    Here are two important methods of BlockingQueue. put(): This method to put objects in queue until queue is full and waits for consumer thread to take object after that.

    take(): This method is used to take objects from the queue until it becomes empty. Once it is empty, it waits for producer threads to start producing the objects and put into the queue.
