    Java String Interview Questions

    1. How to declare a string in Java?

    String declaration in Java can be done in two ways:

    1. By string literal: Double quotes are used to create Java String literals.

      2. Example: String str = "Scaler";

    2. By new keyword: The new keyword is used to create a Java String object.

      3. Example: String str = new String("Scaler");


    2. Is String a primitive or derived type in Java?

    Strings are derived data types. Strings are Java objects that represent sequences of characters.

    String objects are created using the java.lang.String class. There are many functions that need to be called upon when processing a string, such as substring(), indexOf(), equals(), toUpperCase(), etc., which primitive types do not have.


    3. State the difference between String in C and String in Java.

    String in C: In C, strings are just arrays of characters, terminated with '\0' (null-terminated). They can be easily manipulated.

    String in Java: Java treats Strings as objects, created using the java.lang.String class. String objects in Java are immutable; you cannot modify their contents. Operations on strings create new string objects rather than modifying the original.


    4. Explain String pool in Java.

    String Pool (SCP): Special storage space in Java heap memory to store unique string objects. When a string is created, Java checks if the same string value exists in the pool. If yes, it returns a reference; otherwise, it adds a new object to the pool.

    Example:

    							      String s1 = "Apple"; // Created in SCP
							      String s2 = "Mango"; // Created in SCP
							      String s3 = "Apple"; // Reuses s1 reference from SCP

    5. Is String immutable or final in Java? What are the benefits?

    String is immutable in Java, meaning it cannot be changed once created. This ensures security, caching benefits, synchronization, and thread safety.


    6. What does the string intern() method do in Java?

    The intern() method returns a canonical representation of a string. It checks if the string is already in the pool; if yes, returns the reference, otherwise adds it to the pool.

    Example:

    							      String str1 = new String("Scaler").intern();
							      String str2 = new String("Scaler").intern();
							      System.out.println(str1 == str2); // true, points to the same reference in SCP

    7. State the difference between String and StringBuffer.

    String: Immutable, created using double quotes or new String().

    StringBuffer: Mutable, thread-safe with synchronized methods (slower).


    8. State the difference between StringBuffer and StringBuilder in Java.

    StringBuffer: Synchronized methods (thread-safe, slower).

    StringBuilder: Non-synchronized methods (not thread-safe, faster).


    9. In Java, how can two strings be compared?

    Strings can be compared in Java using methods like equals(), equalsIgnoreCase(), compareTo(), and == (for reference comparison).


    10. What is the difference between str1 == str2 and str1.equals(str2)?

    == operator: Compares object references (addresses).

    equals() method: Compares object contents (values).

    Example:

    							      String str1 = new String("Scaler");
							      String str2 = new String("Scaler");
							      System.out.println(str1 == str2); // false
							      System.out.println(str1.equals(str2)); // true

    11. Is it possible to compare Strings using the == operator? If so, what is the risk involved?

    Yes, you can compare strings using the == operator. The == operator in Java compares references (addresses in memory) of objects. When comparing strings with ==, you're comparing whether the two references point to the same object in memory.

    The risk involved is that strings created with different methods (like using the new keyword) or at different times might not point to the same object, even if they have the same content. To compare the actual content of strings, you should use the equals() method, which checks if two strings have the same sequence of characters.

    							        
							public class StringComparison {
							    public static void main(String args[]) {
							        String str1 = "Scaler";
							        String str2 = "Scaler";
							        String str3 = new String("Scaler");
							        
							        System.out.println(str1 == str2);  // true, because both point to the same memory allocation
							        System.out.println(str1 == str3);  // false, because str3 refers to an instance created in heap
							        System.out.println(str1.equals(str3));  // true, because both share the same content
							    }
							}

    When called with one parameter (beginIndex), substring() returns a substring from beginIndex to the end of the string. When called with two parameters (beginIndex, endIndex), it returns a substring that starts at beginIndex and ends just before endIndex.

    							        
							public class InterviewBit {
							    public static void main(String args[]) {
							        String str = "Scaler by InterviewBit";
							        
							        System.out.println(str.substring(7));    // Returns: "by InterviewBit"
							        System.out.println(str.substring(0, 6)); // Returns: "Scaler"
							        System.out.println(str.substring(10, 22));  // Returns: "InterviewBit"
							    }
							}

    15. Is String thread-safe in Java?

    Strings in Java are immutable objects, meaning they cannot be changed or altered once created. Because of this immutability, strings are inherently thread-safe in Java.

    When you manipulate a string (e.g., using methods like substring() or toLowerCase()), a new string object is created rather than modifying the original string. This ensures that multiple threads can safely access and manipulate string objects concurrently without affecting each other.


    16. Why is a string used as a HashMap key in Java?

    In Java, strings are commonly used as keys in HashMap due to their immutability and reliable implementation of hashCode().

    When storing key-value pairs in a HashMap, the hash code of the key is calculated and used to determine the bucket where the key-value pair will be stored. Since strings are immutable, their hash code remains consistent throughout their lifecycle, ensuring that the correct bucket is always identified for retrieval.

    Using strings as keys also ensures efficient retrieval and comparison operations, making them ideal for scenarios where key uniqueness and stability are crucial.

    18. Why is a char array preferred over a String in storing passwords?

    Storing passwords as char arrays instead of strings is preferred in Java due to security concerns and immutability differences.

    • Immutability: Strings are immutable, meaning their values cannot be changed once created. Any modification to a string results in the creation of a new string object, potentially leaving sensitive data in memory longer than necessary.
    • Security: Since strings are immutable, their values are stored in the string constant pool and may linger in memory until garbage collected. This exposes passwords to potential unauthorized access through memory dumps or other means.
    • Logfile Safety: Char arrays allow for overwriting sensitive data after use, reducing the risk of accidental exposure in logs or other insecure locations.

    							            public class StringPermutations {
							    public static void main(String[] args) {
							        String str = "abc";
							        permute(str, 0, str.length() - 1);
							    }
							
							    private static void permute(String str, int left, int right) {
							        if (left == right) {
							            System.out.println(str);
							        } else {
							            for (int i = left; i <= right; i++) {
							                str = swap(str, left, i);
							                permute(str, left + 1, right);
							                str = swap(str, left, i); // Backtrack
							            }
							        }
							    }
							
							    private static String swap(String str, int i, int j) {
							        char[] charArray = str.toCharArray();
							        char temp = charArray[i];
							        charArray[i] = charArray[j];
							        charArray[j] = temp;
							        return new String(charArray);
							    }
							}

    22. Write a program to check whether given string has valid parenthesis or not.

    You can check if a string has valid parentheses using a stack:

    							            public class ValidParentheses {
							    public static void main(String[] args) {
							        String str = "{[()]}";
							        if (isValid(str)) {
							            System.out.println("Valid parentheses.");
							        } else {
							            System.out.println("Invalid parentheses.");
							        }
							    }
							
							    private static boolean isValid(String str) {
							        Stack<Character> stack = new Stack<>();
							        for (char ch : str.toCharArray()) {
							            if (ch == '(' || ch == '[' || ch == '{') {
							                stack.push(ch);
							            } else if (ch == ')' && !stack.isEmpty() && stack.peek() == '(') {
							                stack.pop();
							            } else if (ch == ']' && !stack.isEmpty() && stack.peek() == '[') {
							                stack.pop();
							            } else if (ch == '}' && !stack.isEmpty() && stack.peek() == '{') {
							                stack.pop();
							            } else {
							                return false; // Unmatched closing parenthesis
							            }
							        }
							        return stack.isEmpty(); // Stack should be empty for valid parentheses
							    }
							}
