    Java 8 Features and Enhancements

    1. Lambda Expressions

    Lambda expressions provide a way to write anonymous functions in a more concise manner. They enable functional programming in Java by treating functions as first-class citizens.

    Key Points:

    • Syntax: (parameters) -> expression or (parameters) -> { statements; }
    • Use Case: Reduces boilerplate code in functional interfaces like Runnable, Callable, Comparator, etc.
    • Functional Interface: An interface with a single abstract method, used as the target for lambda expressions.
    // Without Lambda
    Runnable runnable1 = new Runnable() {
      @Override
      public void run() {
        System.out.println("Hello from a thread!");
      }
    };

    // With Lambda
    Runnable runnable2 = () -> System.out.println("Hello from a thread!");

    new Thread(runnable2).start();

    2. Functional Interfaces

    Java 8 introduced several new functional interfaces in the java.util.function package. Additionally, interfaces can now contain default methods with implementations.

    Predicate

    Purpose: The Predicate functional interface represents a boolean-valued function of one argument. It is used to evaluate a condition or test a predicate on a given input.

    Common Methods:

    • boolean test(T t): Evaluates the predicate on the given argument.
    • Predicate and(Predicate other): Returns a composed predicate that represents a short-circuiting logical AND of this predicate and another.
    • Predicate or(Predicate other): Returns a composed predicate that represents a short-circuiting logical OR of this predicate and another.
    • Predicate negate(): Returns a predicate that is the negation of this predicate.

    Example:

    // Import necessary classes
    import java.util.Arrays;
    import java.util.List;
    import java.util.function.Predicate;
    import java.util.stream.Collectors;

    public class PredicateExample {
      public static void main(String[] args) {
        List names = Arrays.asList(
          "John", "Jane", "Jack", "Jill", "Tom");

        Predicate startsWithJ = name -> name.startsWith("J");

        List filteredNames = names.stream()
          .filter(startsWithJ)
          .collect(Collectors.toList());

        System.out.println(filteredNames); // Output: [John, Jane, Jack, Jill]
      }
    }

    Function

    Purpose: The Function functional interface represents a function that takes one argument and produces a result. It is used for transforming or mapping data.

    Common Methods:

    • R apply(T t): Applies this function to the given argument.
    • Function compose(Function before): Returns a composed function that first applies the before function to its input, and then applies this function to the result.
    • Function andThen(Function after): Returns a composed function that first applies this function to its input, and then applies the after function to the result.

    Example:

    // Import necessary classes
    import java.util.Arrays;
    import java.util.List;
    import java.util.function.Function;
    import java.util.stream.Collectors;

    public class FunctionExample {
      public static void main(String[] args) {
        List numbers = Arrays.asList(1, 2, 3, 4, 5);

        Function toString = Object::toString;

        List strings = numbers.stream()
          .map(toString)
          .collect(Collectors.toList());

        System.out.println(strings); // Output: [1, 2, 3, 4, 5]
      }
    }

    Consumer

    Purpose: The Consumer functional interface represents an operation that takes a single argument and returns no result. It is typically used for performing operations like printing, accumulating, or modifying data.

    Common Methods:

    • void accept(T t): Performs the operation on the given argument.
    • Consumer andThen(Consumer after): Returns a composed consumer that performs, in sequence, this operation followed by the after operation.

    Example:

    // Import necessary classes
    import java.util.Arrays;
    import java.util.List;
    import java.util.function.Consumer;

    public class ConsumerExample {
      public static void main(String[] args) {
        List items = Arrays.asList("apple", "banana", "cherry");

        Consumer printConsumer = System.out::println;

        items.forEach(printConsumer);
    }
    }

    Supplier

    Purpose: The Supplier functional interface represents a supplier of results. It provides a result of a given type and is typically used for generating or supplying data.

    Common Methods:

    • T get(): Provides a result of the given type.

    Example:

    // Import necessary classes
    import java.util.Arrays;
    import java.util.List;
    import java.util.function.Supplier;

    public class SupplierExample {
      public static void main(String[] args) {
        Supplier defaultSupplier = () -> "default";

        List defaults = Arrays.asList(
          defaultSupplier.get(),
          defaultSupplier.get(),
          defaultSupplier.get()
        );

        System.out.println(defaults); // Output: [default, default, default]
    }
    }

    3. Stream API

    The Stream API provides a functional approach to processing sequences of elements, such as collections. It supports operations like filtering, mapping, and reducing, enabling easy and efficient data manipulation.

    Key Points:

    • Streams: Support sequential and parallel processing.
    • Operations: Include intermediate (e.g., map, filter, sorted) and terminal (e.g., collect, forEach, reduce) operations.
    • Laziness: Streams are lazy; operations are not performed until a terminal operation is invoked.
    import java.util.Arrays;
    import java.util.List;
    import java.util.stream.Collectors;

    public class StreamExample {
      public static void main(String[] args) {
        List<String> names = Arrays.asList("John", "Jane", "Jack", "Jill");

        List<String> upperCaseNames = names.stream()
          .map(String::toUpperCase)
          .collect(Collectors::toList);

        System.out.println(upperCaseNames);
      }
    }

    4. Method References

    Method references provide a shorthand syntax for calling a method via a lambda expression. They improve code readability and reusability.

    Key Points:

    • Types:
      • Static method reference: ClassName::methodName
      • Instance method reference: instance::methodName
      • Constructor reference: ClassName::new
    • Use Case: Simplifies lambda expressions that only call existing methods.
    import java.util.Arrays;
    import java.util.List;

    public class MethodReferenceExample {
      public static void main(String[] args) {
        List<String> names = Arrays.asList("John", "Jane", "Jack", "Jill");

        names.forEach(System::out.println);
      }
    }

    5. Optional Class

    The Optional class is a container object that may or may not contain a non-null value. It helps to avoid NullPointerException by providing methods to deal with absent values more explicitly.

    Key Points:

    • Methods: isPresent(), ifPresent(), orElse(), orElseGet(), orElseThrow(), etc.
    • Use Case: Eliminates the need for null checks and reduces the risk of NullPointerException.
    import java.util.Optional;

    public class OptionalExample {
      public static void main(String[] args) {
        Optional<String> optional = Optional.ofNullable(null);

        String value = optional.orElse("Default Value");
        System.out.println(value); // Prints: Default Value
      }
    }

    6. New Date and Time API

    Java 8 introduced a new date and time API under the java.time package, which is more comprehensive and easier to work with compared to the old Date and Calendar classes.

    Key Points:

    • Immutable Classes: LocalDate, LocalTime, LocalDateTime, ZonedDateTime, etc.
    • Fluent API: Methods like plusDays(), minusMonths(), withYear(), etc., provide a fluent and readable API.
    • Time Zones: ZoneId, ZoneOffset, and ZonedDateTime handle time zone conversions.
    import java.time.LocalDate;
    import java.time.LocalTime;
    import java.time.LocalDateTime;

    public class DateTimeExample {
      public static void main(String[] args) {
        LocalDate date = LocalDate.now();
        LocalTime time = LocalTime.now();
        LocalDateTime dateTime = LocalDateTime.now();

        System.out.println("Date: " + date);
        System.out.println("Time: " + time);
        System.out.println("DateTime: " + dateTime);
      }
    }

    7. Default and Static Methods in Interfaces

    Java 8 allows interfaces to have default and static methods, which helps in providing method implementations within interfaces.

    Key Points:

    • Default Methods: Methods with a default implementation in an interface.
    • Static Methods: Methods that belong to the interface itself rather than to an instance.
    interface MyInterface {
      // Default method
      default void defaultMethod() {
        System.out.println("This is a default method.");
      }

      // Static method
      static void staticMethod() {
        System.out.println("This is a static method.");
      }
    }

    public class DefaultStaticMethodsExample implements MyInterface {
      public static void main(String[] args) {
        DefaultStaticMethodsExample obj = new DefaultStaticMethodsExample();

        // Calling default method
        obj.defaultMethod();

        // Calling static method
    MyInterface.staticMethod();
      }
    }

    8. Nashorn JavaScript Engine

    Java 8 introduced the Nashorn JavaScript engine, which allows you to run JavaScript code from within Java applications.

    Key Points:

    • Scripting: Execute JavaScript code within Java using the ScriptEngine API.
    • Interoperability: Seamless integration between Java and JavaScript.
    import javax.script.ScriptEngine;
    import javax.script.ScriptEngineManager;
    import javax.script.ScriptException;

    public class NashornExample {
      public static void main(String[] args) throws ScriptException {
        ScriptEngineManager manager = new ScriptEngineManager();
        ScriptEngine engine = manager.getEngineByName("nashorn");

        // Evaluate JavaScript code
        engine.eval("print('Hello from JavaScript')");
      }
    }

    9. Parallel Arrays and Parallel Streams

    Java 8 enhanced the java.util.Arrays class with methods like parallelSort, and it introduced the concept of parallel streams, allowing for parallel processing of data.

    Key Points:

    • Parallel Arrays: Arrays.parallelSort() sorts large arrays in parallel, improving performance on multi-core processors.
    • Parallel Streams: Streams can be processed in parallel using the parallel() method, enabling parallelism with minimal code changes.
    import java.util.Arrays;

    public class ParallelSortExample {
      public static void main(String[] args) {
        int[] numbers = {5, 3, 1, 4, 2};

        // Parallel sort
        Arrays.parallelSort(numbers);

        System.out.println(Arrays.toString(numbers));
      }
    }

    10. Collectors and Collectors API

    The Collectors utility class provides various methods to collect the results of stream operations into different types of collections, such as lists, sets, or maps.

    Key Points:

    • Collectors.toList(): Collects stream elements into a List.
    • Collectors.toSet(): Collects stream elements into a Set.
    • Collectors.groupingBy(): Groups stream elements by a classifier function.
    • Collectors.joining(): Concatenates stream elements into a single string.
    import java.util.Arrays;
    import java.util.List;
    import java.util.Map;
    import java.util.stream.Collectors;

    public class CollectorsExample {
      public static void main(String[] args) {
        List<String> names = Arrays.asList("John", "Jane", "Jack", "Jill");

        // Group names by their length
        Map<Integer, List<String>> groupedByLength = names.stream()
          . collect(Collectors::groupingBy(String::length));

        System.out.println(groupedByLength);
      }
    }

    11. Annotations Enhancements

    Java 8 introduced several enhancements to annotations, including the ability to use annotations on types and repeating annotations.

    Key Points:

    • Type Annotations: Annotations can now be applied to any use of a type, not just declarations.
    • Repeating Annotations: Allows multiple annotations of the same type on a single element.
    import java.lang.annotation.Repeatable;

    @Repeatable(MyAnnotations.class)
    interface MyAnnotation {
      String value();
    }

    interface MyAnnotations {
      MyAnnotation[] value();
    }

    @MyAnnotation("First")
    @MyAnnotation("Second")
    public class RepeatingAnnotationsExample {
      public static void main(String[] args) {
        MyAnnotation[] annotations = RepeatingAnnotationsExample.class.getAnnotationsByType(MyAnnotation.class);
        for (MyAnnotation annotation : annotations) {
          System.out.println(annotation.value());
        }
      }
    }

    12. Intermediate Operations

    Intermediate operations return a new stream and are lazy; they are not executed until a terminal operation is invoked. They can be chained together to form a processing pipeline.

    filter(Predicate<T> predicate):

    • Purpose: Filters elements based on a predicate. Only elements that match the predicate are included in the resulting stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    List<String> filteredNames = names.stream()
      .filter(name -> name.startsWith("J"))
      .collect(Collectors.toList()); // ["John", "Jane", "Jack"]

    map(Function<T, R> mapper):

    • Purpose: Transforms each element using the provided function, producing a stream of transformed elements.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    List<Integer> nameLengths = names.stream()
      .map(String::length)
      .collect(Collectors.toList()); // [4, 4, 4]

    flatMap(Function<T, Stream<R>> mapper):

    • Purpose: Flattens nested streams into a single stream. Each element is mapped to a stream, which is then flattened into a single stream.
    List<List<String>> lists = Arrays.asList(
       Arrays.asList("a", "b"),
       Arrays.asList("c", "d")
    );
    List<String> flattened = lists.stream()
      .flatMap(Collection::stream)
      .collect(Collectors.toList()); // ["a", "b", "c", "d"]

    distinct():

    • Purpose: Removes duplicate elements from the stream based on equals() method.
    List<String> names = Arrays.asList("John", "Jane", "John");
    List<String> distinctNames = names.stream()
      .distinct()
      .collect(Collectors.toList()); // ["John", "Jane"]

    sorted():

    • Purpose: Sorts elements in natural order. Can also be used with a comparator to sort in custom order.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    List<String> sortedNames = names.stream()
      .sorted()
      .collect(Collectors.toList()); // ["Jack", "Jane", "John"]

    limit(long maxSize):

    • Purpose: Limits the stream to the first maxSize elements.
    List<String> names = Arrays.asList("John", "Jane", "Jack", "Jill");
    List<String> limitedNames = names.stream()
      .limit(2)
      .collect(Collectors.toList()); // ["John", "Jane"]

    skip(long n):

    • Purpose: Skips the first n elements of the stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack", "Jill");
    List<String> skippedNames = names.stream()
      .skip(2)
      .collect(Collectors.toList()); // ["Jack", "Jill"]

    peek(Consumer<T> action):

    • Purpose: Performs an action on each element without modifying the stream. Useful for debugging.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    names.stream()
      .peek(name -> System.out.println("Name: " + name))
      .collect(Collectors.toList());

    takeWhile(Predicate<T> predicate) (Java 9+):

    • Purpose: Takes elements from the stream as long as the predicate is true.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
    List<Integer> taken = numbers.stream()
      .takeWhile(n -> n < 4)
      .collect(Collectors.toList()); // [1, 2, 3]

    dropWhile(Predicate<T> predicate) (Java 9+):

    • Purpose: Drops elements from the stream as long as the predicate is true.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
    List<Integer> dropped = numbers.stream()
      .dropWhile(n -> n < 4)
      .collect(Collectors.toList()); // [4, 5]

    13. Terminal Operations

    Terminal operations produce a result or a side-effect and terminate the stream pipeline. They trigger the processing of the elements in the stream.

    forEach(Consumer<T> action):

    • Purpose: Iterates over each element and performs the given action.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    names.stream().forEach(System::out.println);

    collect(Collector<? super T, A, R> collector):

    • Purpose: Accumulates the elements into a collection or other result containers. Common collectors include Collectors.toList(), Collectors.toSet(), and Collectors.toMap().
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    List<String> collectedNames = names.stream()
      .collect(Collectors.toList()); // ["John", "Jane", "Jack"]

    reduce(T identity, BinaryOperator<T> accumulator):

    • Purpose: Reduces the elements to a single result using an associative accumulation function and an identity value.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4);
    int sum = numbers.stream().reduce(0, Integer::sum); // 10

    reduce(BinaryOperator<T> accumulator):

    • Purpose: Reduces the elements to a single result using an associative accumulation function. Returns an Optional if the stream is not empty.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4);
    Optional<Integer> sum = numbers.stream().reduce(Integer::sum); // Optional[10]

    count():

    • Purpose: Counts the number of elements in the stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    long count = names.stream().count(); // 3

    anyMatch(Predicate<T> predicate):

    • Purpose: Checks if any element matches the given predicate. Returns true if any element matches, otherwise false.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    boolean hasNameWithJ = names.stream().anyMatch(name -> name.startsWith("J")); // true

    allMatch(Predicate<T> predicate):

    • Purpose: Checks if all elements match the given predicate. Returns true if all elements match, otherwise false.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    boolean allNamesStartWithJ = names.stream().allMatch(name -> name.startsWith("J")); // false

    noneMatch(Predicate<T> predicate):

    • Purpose: Checks if no elements match the given predicate. Returns true if no elements match, otherwise false.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    boolean noNamesStartWithX = names.stream().noneMatch(name -> name.startsWith("X")); // true

    findFirst():

    • Purpose: Returns an Optional describing the first element of the stream, if present.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Optional<String> firstName = names.stream().findFirst(); // Optional[John]

    findAny():

    • Purpose: Returns an Optional describing some element of the stream, if present. In parallel streams, this might not be the first element.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Optional<String> anyName = names.stream().findAny(); // Optional[John] (or any other element in parallel streams)

    max(Comparator<? super T> comparator):

    • Purpose: Returns the maximum element of the stream according to the given comparator.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4);
    Optional<Integer> max = numbers.stream().max(Integer::compareTo); // Optional[4]

    min(Comparator<? super T> comparator):

    • Purpose: Returns the minimum element of the stream according to the given comparator.
    List<Integer> numbers = Arrays.asList(1, 2, 3, 4);
    Optional<Integer> min = numbers.stream().min(Integer::compareTo); // Optional[1]

    toArray():

    • Purpose: Collects the elements into an array.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    String[] nameArray = names.stream().toArray(String[]::new); // ["John", "Jane", "Jack"]

    14. Collectors Methods

    The Collectors class in Java 8 is a utility class that provides various implementations of the Collector interface. These implementations can be used to accumulate elements from a stream into collections, perform reductions, and more. Heres a detailed overview of the methods in the Collectors class:

    1. toList()

    • Purpose: Collects the elements of a stream into a List.
    List<String> list = Stream.of("a", "b", "c").collect(Collectors.toList()); // ["a", "b", "c"]

    2. toSet()

    • Purpose: Collects the elements of a stream into a Set.
    Set<String> set = Stream.of("a", "b", "c", "a").collect(Collectors.toSet()); // ["a", "b", "c"]

    3. toMap(Function keyMapper, Function valueMapper)

    • Purpose: Collects the elements into a Map, using provided functions to generate keys and values.
    Map<String, Integer> map = Stream.of("a", "bb", "ccc")
    .collect(Collectors.toMap(String::toUpperCase, String::length));
    // {"A"=1, "BB"=2, "CCC"=3}

    4. toMap(Function keyMapper, Function valueMapper, BinaryOperator mergeFunction)

    • Purpose: Collects the elements into a Map, allowing for a merge function to handle duplicate keys.
    Map<String, Integer> map = Stream.of("a", "b", "a")
    .collect(Collectors.toMap(Function::identity, String::length, (existing, replacement) -> existing));
    // {"a"=1, "b"=1}

    5. joining(CharSequence delimiter, CharSequence prefix, CharSequence suffix)

    • Purpose: Concatenates the elements of a stream into a single String, with optional delimiter, prefix, and suffix.
    String result = Stream.of("a", "b", "c")
    .collect(Collectors.joining(", ", "[", "]")); // "[a, b, c]"

    6. groupingBy(Function classifier)

    • Purpose: Groups the elements by a classifier function. Returns a Map where keys are the result of applying the classifier function, and values are lists of items.
    Map<Integer, List<String>> groupedByLength = Stream.of("a", "bb", "ccc")
    .collect(Collectors.groupingBy(String::length));
    // {1=["a"], 2=["bb"], 3=["ccc"]}

    7. groupingBy(Function classifier, Collector downstream)

    • Purpose: Groups the elements by a classifier function, and then applies a downstream collector to the grouped values.
    Map<Integer, Set<String>> groupedByLength = Stream.of("a", "bb", "ccc")
    .collect(Collectors.groupingBy(String::length, Collectors.toSet()));
    // {1=[a], 2=[bb], 3=[ccc]}

    8. partitioningBy(Predicate predicate)

    • Purpose: Partitions the elements into two groups based on a predicate. Returns a Map with Boolean keys.
    Map<Boolean, List<String>> partitioned = Stream.of("a", "bb", "ccc")
    .collect(Collectors.partitioningBy(s -> s.length() > 1));
    // {false=[a], true=[bb, ccc]}

    9. counting()

    • Purpose: Counts the number of elements in the stream.
    long count = Stream.of("a", "b", "c")
    .collect(Collectors.counting());
    // 3

    10. summingInt(ToIntFunction mapper)

    • Purpose: Sums the integer values produced by applying the provided function to each element.
    int sum = Stream.of("a", "bb", "ccc")
    .collect(Collectors.summingInt(String::length));
    // 6

    11. averagingInt(ToIntFunction mapper)

    • Purpose: Averages the integer values produced by applying the provided function to each element.
    double average = Stream.of("a", "bb", "ccc")
    .collect(Collectors.averagingInt(String::length));
    // 2.0

    12. summarizingInt(ToIntFunction mapper)

    • Purpose: Produces an IntSummaryStatistics describing various statistics, including count, sum, min, average, and max.
    IntSummaryStatistics stats = Stream.of("a", "bb", "ccc")
    .collect(Collectors.summarizingInt(String::length));
    // count=3, sum=6, min=1, average=2.0, max=3

    13. mapping(Function mapper, Collector downstream)

    • Purpose: Applies a mapping function to each element and then collects the result with a downstream collector.
    List<Integer> result = Stream.of("a", "bb", "ccc")
    .collect(Collectors.mapping(String::length, Collectors.toList()));
    // [1, 2, 3]

    14. reducing(identity, BinaryOperator)

    • Purpose: Performs a reduction on the elements using an identity value and an associative accumulation function.
    int sum = Stream.of(1, 2, 3)
    .collect(Collectors.reducing(0, Integer::sum));
    // 6

    15. Other Stream Methods

    In addition to the Collectors methods, the Stream interface offers a wide array of methods for processing streams. Heres a detailed overview:

    1. filter(Predicate predicate)

    • Purpose: Returns a stream consisting of the elements that match the given predicate.
    List<String> filtered = Stream.of("a", "bb", "ccc")
    .filter(s -> s.length .collect(Collectors.toList());
    // ["bb", "ccc"]

    2. map(Function mapper)

    • Purpose: Returns a stream consisting of the results of applying the given function to the elements.
    List<Integer> mapped = Stream.of("a", "bb", "ccc")
    .map(String::length)
    .collect(Collectors.toList());
    // [1, 2, 3]

    3. flatMap(Function> mapper)

    • Purpose: Returns a stream that is the result of replacing each element of this stream with the contents of a mapped stream produced by applying the given function.
    List<String> flatMapped = Stream.of("a", "bb", "ccc")
    .flatMap(s -> Stream.of(s.split(""))
    .collect(Collectors.toList());
    // [a, b, b, c, c, c]

    4. distinct()

    • Purpose: Returns a stream consisting of distinct elements.
    List<Integer> distinct = Stream.of(1, 2, 2, 3, 3, 3)
    .distinct()
    .collect(Collectors.toList());
    // [1, 2, 3]

    5. sorted()

    • Purpose: Returns a stream consisting of the elements sorted according to their natural order or by a provided comparator.
    List<Integer> sorted = Stream.of(3, 1, 2)
    .sorted()
    .collect(Collectors.toList());
    // [1, 2, 3]

    6. limit(long maxSize)

    • Purpose: Returns a stream consisting of the elements of this stream, truncated to be no longer than maxSize in length.
    List<Integer> limited = Stream.of(1, 2, 3, 4, 5)
    .limit(3)
    .collect(Collectors.toList());
    // [1, 2, 3]

    7. skip(long n)

    • Purpose: Returns a stream consisting of the remaining elements of this stream after discarding the first n elements.
    List<Integer> skipped = Stream.of(1, 2, 3, 4, 5)
    .skip(2)
    .collect(Collectors.toList());
    // [3, 4, 5]

    8. forEach(Consumer action)

    • Purpose: Performs an action for each element of the stream.
    Stream.of("a", "b", "c")
    .forEach(System::out.println);
    // prints a b c

    9. peek(Consumer action)

    • Purpose: Performs an action on each element as it is consumed from the stream. Useful for debugging.
    List<Integer> peeked = Stream.of(1, 2, 3)
    .peek(System::out.println)
    .collect(Collectors.toList());
    // prints 1, 2, 3 and returns [1, 2, 3]

    10. findFirst()

    • Purpose: Returns an Optional describing the first element of this stream, or an empty Optional if the stream is empty.
    Optional<String> first = Stream.of("a", "b", "c")
    .findFirst();
    // Optional[a]

    11. reduce(BinaryOperator accumulator)

    • Purpose: Performs a reduction on the elements of the stream using an associative accumulation function and returns an Optional describing the result.
    Optional<Integer> sum = Stream.of(1, 2, 3)
    .reduce(Integer::sum);
    // Optional[6]

    12. toArray()

    • Purpose: Returns an array containing the elements of this stream.
    String[] array = Stream.of("a", "b", "c")
    .toArray(String[]::new);
    // [a, b, c]

    13. iterator()

    • Purpose: Returns an iterator over the elements of the stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Iterator<String> iterator = names.stream().iterator();
    while (iterator.hasNext()) {
      System.out.println(iterator.next());
    }

    14. spliterator()

    • Purpose: Returns a Spliterator for the elements of the stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Spliterator<String> spliterator = names.stream().spliterator();
    spliterator.forEachRemaining(System::out.println);

    15. isParallel()

    • Purpose: Returns true if the stream is parallel, false otherwise.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    boolean isParallel = names.stream().isParallel();
    // false

    16. sequential()

    • Purpose: Returns a sequential stream with the same elements as the current stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Stream<String> sequentialStream = names.stream().parallel().sequential();

    17. parallel()

    • Purpose: Returns a parallel stream with the same elements as the current stream.
    List<String> names = Arrays.asList("John", "Jane", "Jack");
    Stream<String> parallelStream = names.stream().parallel();

    18. close()

    • Purpose: Closes the stream and releases any resources associated with it.
    try (Stream<String> stream = Files.lines(Paths.get("file.txt"))) {
      stream.forEach(System::out.println);
    } catch (IOException e) {
      e.printStackTrace();
    }

    These methods provide powerful ways to manipulate and process data in Java streams, enabling you to handle collections and sequences efficiently.
